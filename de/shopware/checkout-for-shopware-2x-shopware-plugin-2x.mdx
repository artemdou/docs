## Übersicht

Unser Shopware-Plugin ist die perfekte Erweiterung, um das volle Potenzial von Checkout für Ihren Shopware-Shop auszuschöpfen. Checkout unterstützt die gängigsten Zahlungsmethoden.
Mit Version 2 haben wir eine Reihe leistungsstarker neuer Funktionen eingeführt:
- Aufteilung der Embedded- und Hosted-Flows in zwei separate Zahlungsoptionen
- Vollständige Webhook-Unterstützung
- Belastungsverwaltung (einschließlich Workflow-Integration – z. B. Charge-Aktion im Workflow)
- Kompatibilität mit Shopware 6.7
- Integration mit unserem SDK (nexi-checkout/php-payment-sdk)
- Unterstützung für PHP 8.4
- Layout-Verbesserungen für ein flüssigeres Checkout-Erlebnis

</Info>

## Installation

So installieren Sie das Checkout-Modul für Shopware 6 von GitHub:

- Verbinden Sie sich mit einem SSH-Client und navigieren Sie zum Stammverzeichnis Ihrer Shopware 6-Installation.

- Klonen Sie das Repository mit dem Befehl: git clone git@github.com:Nets-eCom/shopware6-easy-checkout.git --branch main-1.5 --single-branch custom/plugins/NexiCheckout

Nach dem Klonen führen Sie diese Befehle aus:
- bin/console plugin:refresh
- bin/console plugin:install NexiCheckout
- bin/console plugin:activate NexiCheckout
- bin/build-js.sh

Das Modul ist nun installiert und kann für Ihr Checkout-Konto konfiguriert werden.

## Konfiguration

Nach der Installation des Moduls müssen Sie einige grundlegende Konfigurationen im Shopware Admin vornehmen:

1. Navigieren Sie zu Erweiterungen > Meine Erweiterungen > NexiNets Checkout
2. Suchen Sie das Nexi Group-Zahlungsplugin und drücken Sie die Schaltfläche mit den drei Punkten (...), um auf die Konfiguration zuzugreifen.
3. Füllen Sie die erforderlichen Felder aus, z. B. Integrationsschlüssel (Secret Keys und Checkout Keys) und Webhook-Code.
4. (Optional) Passen Sie das Modul mit den zusätzlichen Einstellungen auf der Konfigurationsseite an Ihre Bedürfnisse an.

Beide Integrationsschlüssel finden Sie im Checkout Portal. Weitere Hilfe finden Sie auf den folgenden Seiten:

[Wo finde ich meine Händlernummer (Merchant ID)?](https://developer.nexigroup.com/nexi-checkout/en-EU/support/where-can-i-find-my-merchant-number-merchant-id/)](https://developer.nexigroup.com/nexi-checkout/en-EU/support/where-can-i-find-my-merchant-number-merchant-id/)
[Auf Ihre Integrationsschlüssel zugreifen](/access-your-integration-keys)

Integrationstyp: Embedded ist noch keine gültige Option. Bitte verwenden Sie Hosted Payment.

## Bestellverwaltung

Es ist möglich, Bestellungen direkt in der Shopware-Verwaltung zu verwalten:

Navigieren Sie zu Admin > Bestellungen > Übersicht.
Drücken Sie auf eine Bestellzeile, um auf die Bestelldetails zuzugreifen.
Gehen Sie zum Abschnitt Details mit der neuen Komponente Nexi Bestellverwaltung.

![order-details](https://images.ctfassets.net/di680may5c0k/2PEOw9hNRzHWIHflneFDye/6bf820ae2748185345a357e33c8872f6/order-details.png)

Nun können alle Aktionen (Belastung, Rückerstattung, Stornierung) über die neue Modal-Komponente durchgeführt werden.

![charge-modal](https://images.ctfassets.net/di680may5c0k/5puVzKOdX3O9sLIabFNLF3/b1cc621230e5f00b2f73f8d1b02feebb/charge-modal.png)

Wenn Sie auf Max. amount klicken, wird der Maximalwert automatisch in das Betragsfeld übernommen.

__Teilbelastung & Teil-Rückerstattung__

Die größte Änderung gegenüber der derzeit verwendeten Komponente besteht darin, dass Teilbelastung und Teil-Rückerstattung durch Auswahl des Artikels aus der Bestellliste vorgenommen werden können, anstatt den Wert manuell in das Feld einzugeben. Auf die Liste kann über das Kontrollkästchen-Feld zugegriffen werden.

## Klarna

Die Zahlungsmethode Klarna erfordert eine Telefonnummer, um ordnungsgemäß zu funktionieren. Damit Klarna im Nexi Group-Zahlungsfenster als Zahlungsoption angezeigt wird, ist es wichtig, das Telefonnummernfeld korrekt zu konfigurieren. Weitere Informationen finden Sie im [Klarna-Leitfaden](/klarna/#build-integration).

So fügen Sie das Telefonnummernfeld in Shopware 6 hinzu:
- Navigieren Sie zu Einstellungen > Shop > Anmeldung & Registrierung.
- Aktivieren Sie das Kontrollkästchen Telefonnummer anzeigen.
- Aktivieren Sie das Kontrollkästchen Telefonnummernfeld erforderlich.

Weitere Informationen finden Sie in der [Shopware 6-Dokumentation](https://docs.shopware.com/en/shopware-en/settings/login-registration).

![shopware klarna contentful](https://images.ctfassets.net/di680may5c0k/7f1JSaWMEGxuUPPi5EVRaR/272c8bc26efb45188bf031d3171c8842/shopware_klarna_contentful.png?w=1273&h=638&q=50&fm=png)

<Info>
Wir unterstützen keine Drittanbieter-Plugins, die Telefonnummernfunktionen bereitstellen, und können nicht garantieren, dass deren Lösung mit der Klarna-Zahlungsmethode kompatibel ist.
</Info>

## Fehlerbehebung

Im Folgenden finden Sie einige der häufigsten Konfigurationsfehler, ihre Ursachen und Schritte, die Sie befolgen können, um sie zu beheben.

### Die Nexi Group-Zahlungs-App ist nicht als Zahlungsmethode sichtbar

- Stellen Sie sicher, dass die Nexi Group-App im richtigen **Sales Channel** in der App-Konfiguration verfügbar ist.
- Wählen Sie unter dem Abschnitt **Sales Channel** Ihren **Shop-Namen** unter den **Allgemeinen** Einstellungen aus. Fügen Sie die gewünschten Zahlungsmethoden aus der Nexi-App hinzu.
- Wechseln Sie vorübergehend zum Shopware 6-Standardtemplate. Benutzerdefinierte Templates erfordern möglicherweise zusätzliche Änderungen, um eine korrekte Anzeige zu gewährleisten. Wenden Sie sich an Ihren Webdesigner oder Entwickler.

### Das Nexi Group-Zahlungsfenster ist leer

- Stellen Sie sicher, dass Ihre Integrationsschlüssel in den Nexi Group-App-Einstellungen korrekt sind und keine zusätzlichen Leerzeichen enthalten.
- Deaktivieren Sie vorübergehend Plugins von Drittanbietern, die die Funktionalität der App beeinträchtigen könnten.
- Prüfen Sie, ob es vorübergehende technische Probleme gibt: https://easy-status.developers.nets.eu/

### Zahlungen im Live-Modus funktionieren nicht
- Stellen Sie sicher, dass Sie ein genehmigtes Live Checkout-Konto für die Produktion haben.
- Stellen Sie sicher, dass Ihr Live Checkout-Konto für Zahlungen mit der gewählten Währung freigegeben ist.
- Stellen Sie sicher, dass die Zahlungsmethodendaten korrekt sind und von Ihrem Checkout-Vertrag unterstützt werden.

## Bevor Sie beginnen

<Warning>
Bevor Sie beginnen, benötigen Sie eine Checkout-Seite, die die [Checkout-Testumgebung](/test-environment) verwendet.
</Warning>

## So testen Sie

Die Checkout-Seite bietet Ihren Kunden die Möglichkeit, mit Karte zu bezahlen. Um Kartenzahlungen auf der Checkout-Seite zu testen, gehen Sie wie folgt vor:

1. Navigieren Sie zu Ihrer Checkout-Seite.
2. Wählen Sie "Mit Karte bezahlen".
3. Fügen Sie eine der Beispielkartennummern aus der Spalte **Nummer** in der unten stehenden Tabelle hinzu.
4. Geben Sie ein zukünftiges Ablaufdatum ein.
5. Geben Sie 3 beliebige Ziffern in das CVC-Feld ein.
6. Geben Sie einen beliebigen Namen, eine Adresse und eine Telefonnummer in das Adressformular ein.

<Warning>
Bei der Verwendung der Beispielkarten in der Testumgebung haben einige Beträge eine besondere Bedeutung und erzeugen Fehlercodes oder erhöhen die API-Antwortzeiten. Dies ist kein Fehler, sondern ein Feature der Testumgebung, damit Sie Ihr System gegen Fehlercodes und verzögerte Antworten testen können. Weitere Informationen finden Sie im folgenden Abschnitt.
</Warning>

## Beträge mit besonderer Bedeutung

Es ist möglich zu testen, wie sich Ihre Website verhält, wenn Checkout verschiedene **Fehlercodes** zurückgibt oder der Checkout-Prozess **verzögert** wird. Die folgenden Beträge haben eine besondere Bedeutung und ändern das Checkout-Verhalten beim Erstellen einer neuen Zahlung in der Testumgebung:

- Der Betrag **0.XX** und **1.XX** löst den **Fehlercode XX** aus. Wenn beispielsweise die Eigenschaft [`order.amount `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post-body-order-amount) auf `133` gesetzt ist, wird der Fehlercode `33` ausgelöst.

- Der Betrag **3.XX** **verzögert** die API-Antwort um **XX Sekunden**. Wenn beispielsweise die Eigenschaft [`order.amount `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post-body-order-amount) auf `310` gesetzt ist, wird die Antwort aus dem Checkout-Ablauf um 10 Sekunden verzögert.

<Info>
Beachten Sie, dass die Beträge in der [niedrigsten Währungseinheit für die jeweilige Währung, ohne Satzzeichen](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api#data-types-and-formats), angegeben werden.
</Info>

Mehr über Begründungscodes erfahren Sie [hier](/reason-codes).

## Rabatte

Es ist möglich, in den Zahlungsanfragen Rabatte anzugeben. Dies geschieht durch Hinzufügen einer Bestellzeile mit einem negativen Betrag.

Ein Beispiel hierfür finden Sie im Payload-Beispiel [hier](/web-integration/integrate-checkout-on-your-website-embedded/#build-step-2-create-a-payment-object-backend).

<Warning>
Bitte beachten Sie, dass der Gesamtbetrag bei den folgenden Anfragen niemals 0 oder kleiner sein darf:
- [unit price](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post-body-order-items-unitprice)
- [gross total amount](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post-body-order-items-grosstotalamount)
- [net total amount](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post-body-order-items-nettotalamount)
- [orderItems](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-paymentid-charges-post-body-orderitems)
- [charge orderItems  ](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-charges-chargeid-refunds-post-body-orderitems)
</Warning>

Wenn die [Create-Payment-Anfrage](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post) `"charge":true ` enthält, schlägt die Belastung mit der Testkarte 4925000000000079 nicht fehl.
</Warning>

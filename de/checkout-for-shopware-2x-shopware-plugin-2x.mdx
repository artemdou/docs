## Bevor Sie beginnen

<Warning>
Bevor Sie beginnen, benötigen Sie ein [Checkout Portal-Konto](https://portal.dibspayment.eu). Bitte beachten Sie die Anleitung:

- Shopware-Version: Kompatibel mit Shopware 6.6.x und höher.

Wenn Sie Shopware 1.x verwenden, müssen Sie Shopware 2.x parallel betreiben, bis die Bestellungen abgeschlossen sind.

## Übersicht

Unser Shopware-Plugin ist die perfekte Erweiterung, um das volle Potenzial von Checkout für Ihren Shopware-Store auszuschöpfen. Checkout unterstützt die beliebtesten Zahlungsmethoden.
Mit Version 2 haben wir eine Reihe leistungsstarker neuer Funktionen eingeführt:
- Aufteilung von Embedded- und Hosted-Flows in zwei separate Zahlungsoptionen
- Vollständige Webhook-Unterstützung
- Charge-Verarbeitung (einschließlich Workflow-Integration – z. B. Charge-Aktion im Workflow)
- Kompatibilität mit Shopware 6.7
- Integration mit unserem SDK (nexi-checkout/php-payment-sdk)
- Unterstützung für PHP 8.4
- Layout-Verbesserungen für ein reibungsloseres Checkout-Erlebnis

## Installation

So installieren Sie das Checkout-Modul für Shopware 6 von GitHub:

- Verbinden Sie sich über einen SSH-Client und navigieren Sie zum Stammverzeichnis Ihrer Shopware 6-Installation.

- Klonen Sie das Repository, indem Sie folgenden Befehl ausführen: git clone git@github.com:Nets-eCom/shopware6-easy-checkout.git --branch main-1.5 --single-branch custom/plugins/NexiCheckout

Führen Sie nach dem Klonen diese Befehle aus:
- bin/console plugin:refresh
- bin/console plugin:install NexiCheckout
- bin/console plugin:activate NexiCheckout
- bin/build-js.sh

Das Modul ist nun installiert und kann für Ihr Checkout-Konto konfiguriert werden.

## Konfiguration

Nach der Installation des Moduls müssen Sie einige grundlegende Konfigurationen des Moduls in der Shopware-Administration vornehmen:

1. Navigieren Sie zu Erweiterungen > Meine Erweiterungen > NexiNets Checkout
2. Suchen Sie das Nexi Group-Zahlungs-Plugin und drücken Sie die Schaltfläche mit den drei Punkten (...), um auf die Konfiguration zuzugreifen.
3. Füllen Sie die erforderlichen Felder aus, wie z. B. Integrationsschlüssel (Secret Keys und Checkout Keys) und Webhook-Code.
4. (Optional) Passen Sie das Modul nach Ihren Bedürfnissen an, indem Sie die zusätzlichen Einstellungen auf der Konfigurationsseite verwenden.

Beide Integrationsschlüssel finden Sie im Checkout Portal. Weitere Hilfe finden Sie auf den folgenden Seiten:

[Wo finde ich meine Händlernummer (Merchant ID)?](https://developer.nexigroup.com/nexi-checkout/en-EU/support/where-can-i-find-my-merchant-number-merchant-id/)](https://developer.nexigroup.com/nexi-checkout/en-EU/support/where-can-i-find-my-merchant-number-merchant-id/)
[Auf Ihre Integrationsschlüssel zugreifen](/access-your-integration-keys)

Integrationstyp: Embedded ist noch keine gültige Option. Verwenden Sie bitte Hosted Payment.

## Auftragsverwaltung

Es ist möglich, Bestellungen direkt in der Shopware-Administration zu verwalten:

Navigieren Sie zu Admin > Bestellungen > Übersicht.
Klicken Sie auf eine Bestellzeile, um auf die Bestelldetails zuzugreifen.
Wechseln Sie zum Bereich „Details" mit der neuen Nexi-Auftragsverwaltungs-Komponente.

![Bestelldetails](https://images.ctfassets.net/di680may5c0k/2PEOw9hNRzHWIHflneFDye/6bf820ae2748185345a357e33c8872f6/order-details.png)

Nun können alle Aktionen (Charge, Refund, Cancel) über die neue modale Komponente durchgeführt werden.

![Charge-Modal](https://images.ctfassets.net/di680may5c0k/5puVzKOdX3O9sLIabFNLF3/b1cc621230e5f00b2f73f8d1b02feebb/charge-modal.png)

Durch Klicken auf „Max. amount" wird automatisch der maximale Wert in das Betragsfeld übernommen.

__Teilbelastung & Teilrückerstattung__

Die größte Änderung gegenüber der derzeit verwendeten Komponente besteht darin, dass Teilbelastungen und Teilrückerstattungen durchgeführt werden können, indem die Position aus der Bestellliste ausgewählt wird, anstatt den Wert manuell in das Feld einzugeben. Auf die Liste kann über das Kontrollkästchen-Feld zugegriffen werden.

![Nexi-Teilbelastungen](https://images.ctfassets.net/di680may5c0k/7M46h7zMlr3BEX7NdMhKX5/c1d66351ef5d43f9ac8327eb0277c1ac/nexi-partials.png)

## Klarna

Die Zahlungsmethode Klarna erfordert eine Telefonnummer, um ordnungsgemäß zu funktionieren. Damit Klarna als Zahlungsoption im Zahlungsfenster der Nexi Group erscheint, muss das Telefonnummernfeld korrekt konfiguriert werden. Weitere Informationen finden Sie in der [Klarna-Anleitung](/klarna/#build-integration).

Um das Telefonnummernfeld in Shopware 6 hinzuzufügen, gehen Sie wie folgt vor:
- Navigieren Sie zu Einstellungen > Shop > Login & Registrierung.
- Aktivieren Sie das Kontrollkästchen für „Telefonnummer anzeigen".
- Aktivieren Sie das Kontrollkästchen für „Telefonnummernfeld erforderlich".

Weitere Informationen finden Sie in der [Shopware 6-Dokumentation](https://docs.shopware.com/en/shopware-en/settings/login-registration).

![Shopware Klarna Contentful](https://images.ctfassets.net/di680may5c0k/7f1JSaWMEGxuUPPi5EVRaR/272c8bc26efb45188bf031d3171c8842/shopware_klarna_contentful.png?w=1273&h=638&q=50&fm=png)

<Info>
Wir unterstützen keine Drittanbieter-Plugins, die Telefonnummernfunktionalität bereitstellen, und können nicht garantieren, dass deren Lösung mit der Klarna-Zahlungsmethode kompatibel ist.
</Info>

## Fehlerbehebung

Nachfolgend finden Sie einige der häufigsten Konfigurationsfehler, ihre Ursachen und Schritte, mit denen Sie sie beheben können.

### Nexi Group Payment App ist nicht als Zahlungsmethode sichtbar

- Stellen Sie sicher, dass die Nexi Group-App im richtigen **Verkaufskanal** in der App-Konfiguration verfügbar ist.
- Wählen Sie unter dem Abschnitt **Verkaufskanal** Ihren **Shop-Namen** unter den **Allgemein**-Einstellungen aus. Fügen Sie die gewünschten Zahlungsmethoden aus der Nexi-App hinzu.
- Wechseln Sie vorübergehend zum Standard-Template von Shopware 6. Benutzerdefinierte Templates erfordern möglicherweise zusätzliche Änderungen, um eine korrekte Anzeige zu gewährleisten. Konsultieren Sie Ihren Webdesigner oder Entwickler.

### Das Zahlungsfenster der Nexi Group ist leer

- Stellen Sie sicher, dass Ihre Integrationsschlüssel in den Einstellungen der Nexi Group-App korrekt sind und keine zusätzlichen Leerzeichen enthalten.
- Deaktivieren Sie vorübergehend Drittanbieter-Plugins, die die Funktionalität der App beeinträchtigen könnten.
- Prüfen Sie, ob temporäre technische Inkonsistenzen vorliegen: https://easy-status.developers.nets.eu/

### Zahlungen im Live-Modus funktionieren nicht
- Stellen Sie sicher, dass Sie ein genehmigtes Live-Checkout-Konto für die Produktion haben.
- Stellen Sie sicher, dass Ihr Live-Checkout-Konto für Zahlungen in der ausgewählten Währung genehmigt ist.
- Stellen Sie sicher, dass die Zahlungsmethodendaten korrekt sind und von Ihrem Checkout-Vertrag unterstützt werden.

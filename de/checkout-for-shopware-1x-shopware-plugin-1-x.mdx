## Bevor Sie beginnen

<Warning>
Bevor Sie beginnen, benötigen Sie ein [Checkout-Portal-Konto](https://portal.dibspayment.eu). Bitte beachten Sie die Anleitung

Kompatibel mit Shopware 6.4 und 6.5

Hinweis: Für Shopware 6.6.x und höher verwenden Sie bitte Shopware 2.x

__Bitte beachten Sie, dass es keinen Upgrade-Pfad von Shopware 1.x zu Shopware 2.x gibt.__

## Übersicht

Unser Shopware-Plugin ist die perfekte Erweiterung, um den Nexi-Checkout in Ihrem Shopware-Shop voll auszuschöpfen. Checkout unterstützt die meisten gängigen Zahlungsmethoden.

<Info>
Nachfolgend sehen Sie alle von Checkout angebotenen Zahlungsmethoden. Diese Liste bezieht sich auf alle Märkte.

Je nach Land oder Region kann die Liste variieren. Wenn Sie bei einer bestimmten Zahlungsmethode unsicher sind, ob diese in Ihrem Land oder Ihrer Region verfügbar ist, kontaktieren Sie bitte den [Support](https://developer.nexigroup.com/nexi-checkout/en-EU/support/) für weitere Informationen.
</Info>

## Installation

__Installation__

So installieren Sie das Checkout-Modul für Shopware 6 von GitHub:

- Verbinden Sie sich mit einem SSH-Client und navigieren Sie in das Stammverzeichnis Ihrer Shopware-6-Installation.

- Klonen Sie das Repository mit dem Befehl: git clone git@github.com:Nets-eCom/shopware6-easy-checkout.git --branch main-1.5 --single-branch custom/plugins/NexiCheckout

Führen Sie nach dem Klonen die folgenden Befehle aus:

- bin/console plugin:refresh
- bin/console plugin:install NexiCheckout
- bin/console plugin:activate NexiCheckout
- bin/build-js.sh

Das Modul ist nun installiert und kann für Ihr Checkout-Konto konfiguriert werden.

## Konfiguration

Nach der Installation des Moduls müssen Sie im Shopware-Admin einige grundlegende Konfigurationen vornehmen:

1. Navigieren Sie im Shopware-Backend/Admin-Panel zu **Erweiterungen > Meine Erweiterungen**.
2. Suchen Sie das Nexi Group Zahlungs-Plugin und klicken Sie auf den Button mit den drei Punkten (...), um auf die Konfiguration zuzugreifen.
3. Füllen Sie die erforderlichen Felder aus, wie z. B. Händler-ID und Integrationsschlüssel (Secret-Keys und Checkout-Keys).
4. (Optional) Passen Sie das Modul mit den zusätzlichen Einstellungen auf der Konfigurationsseite Ihren Bedürfnissen an.

<Info>
Sowohl die **Händler-ID** als auch die **Integrationsschlüssel** finden Sie im [Checkout Portal](https://portal.dibspayment.eu/). Weitere Informationen finden Sie auf den folgenden Seiten:

- [Wo finde ich meine Händlernummer (Merchant-ID)?](https://developer.nexigroup.com/nexi-checkout/en-EU/support/where-can-i-find-my-merchant-number-merchant-id/)
- [Auf Ihre Integrationsschlüssel zugreifen](/access-your-integration-keys)
</Info>

## Auftragsverwaltung

Es ist möglich, Bestellungen direkt in der Shopware-Administration zu verwalten:

Navigieren Sie zu Admin > Bestellungen > Übersicht.
Klicken Sie auf eine Bestellzeile, um die Bestelldetails aufzurufen.
Wechseln Sie zum Bereich „Details“ mit der neuen Nexi-Auftragsverwaltungs-Komponente.

![order-details](https://images.ctfassets.net/di680may5c0k/2PEOw9hNRzHWIHflneFDye/6bf820ae2748185345a357e33c8872f6/order-details.png)

Nun können alle Aktionen (Charge, Refund, Cancel) über die neue Modal-Komponente ausgeführt werden.

![charge-modal](https://images.ctfassets.net/di680may5c0k/5puVzKOdX3O9sLIabFNLF3/b1cc621230e5f00b2f73f8d1b02feebb/charge-modal.png)

Ein Klick auf „Max. Betrag“ überträgt automatisch den Maximalwert in das Betragsfeld.

__Teilweise Einziehung & Erstattung__

Die größte Änderung gegenüber der bisher verwendeten Komponente besteht darin, dass teilweise Einziehungen und Erstattungen durch Auswahl des Artikels aus der Bestellliste vorgenommen werden können, anstatt den Betrag manuell in das Feld einzugeben. Die Liste lässt sich über das Kontrollkästchenfeld aufrufen.

![nexi-partials](https://images.ctfassets.net/di680may5c0k/7M46h7zMlr3BEX7NdMhKX5/c1d66351ef5d43f9ac8327eb0277c1ac/nexi-partials.png)

## Klarna

Die Zahlungsmethode Klarna benötigt eine Telefonnummer, um ordnungsgemäß zu funktionieren. Damit Klarna im Nexi Group Zahlungsfenster als Zahlungsoption erscheint, muss das Telefonnummernfeld korrekt konfiguriert werden. Weitere Informationen finden Sie im [Klarna-Leitfaden](/klarna/#build-integration).

So fügen Sie das Telefonnummernfeld in Shopware 6 hinzu:
- Navigieren Sie zu Einstellungen > Shop > Anmeldung & Registrierung.
- Aktivieren Sie das Kontrollkästchen für „Telefonnummer anzeigen“.
- Aktivieren Sie das Kontrollkästchen für „Telefonnummernfeld erforderlich“.

Weitere Informationen finden Sie in der [Shopware 6-Dokumentation](https://docs.shopware.com/en/shopware-en/settings/login-registration).

![shopware klarna contentful](https://images.ctfassets.net/di680may5c0k/7f1JSaWMEGxuUPPi5EVRaR/272c8bc26efb45188bf031d3171c8842/shopware_klarna_contentful.png?w=1273&h=638&q=50&fm=png)

<Info>
Wir unterstützen keine Drittanbieter-Plugins, die eine Telefonnummern-Funktionalität bereitstellen, und können nicht garantieren, dass deren Lösung mit der Klarna-Zahlungsmethode kompatibel ist.
</Info>

## Fehlerbehebung

Nachfolgend finden Sie einige der häufigsten Konfigurationsfehler, deren Ursachen und Schritte zu deren Behebung.

### Nexi Group Zahlungs-App ist nicht als Zahlungsmethode sichtbar

- Stellen Sie sicher, dass die Nexi Group App im richtigen **Sales Channel** in der App-Konfiguration verfügbar ist.
- Wählen Sie unter dem Abschnitt **Sales Channel** Ihren **Shop-Namen** unter den **Allgemeinen** Einstellungen aus. Fügen Sie die gewünschten Zahlungsmethoden aus der Nexi-App hinzu.
- Wechseln Sie vorübergehend zum Standard-Template von Shopware 6. Benutzerdefinierte Templates erfordern möglicherweise zusätzliche Anpassungen, um eine korrekte Anzeige sicherzustellen. Wenden Sie sich an Ihren Webdesigner oder Entwickler.

### Nexi Group Zahlungsfenster ist leer

- Stellen Sie sicher, dass Ihre Integrationsschlüssel in den Nexi Group App-Einstellungen korrekt sind und keine zusätzlichen Leerzeichen enthalten.
- Deaktivieren Sie vorübergehend Drittanbieter-Plugins, die die Funktionalität der App beeinträchtigen könnten.
- Prüfen Sie, ob es vorübergehende technische Inkonsistenzen gibt: https://easy-status.developers.nets.eu/

### Zahlungen im Live-Modus funktionieren nicht
- Stellen Sie sicher, dass Sie über ein genehmigtes Live-Checkout-Konto für die Produktion verfügen.
- Stellen Sie sicher, dass Ihr Live-Checkout-Konto für Zahlungen in der gewählten Währung freigegeben ist.
- Stellen Sie sicher, dass die Zahlungsmethodendaten korrekt sind und durch Ihren Checkout-Vertrag unterstützt werden.

## Übersicht

Unser Plugin für Magento 2 unterstützt alle Checkout-Funktionen wie smarte Kundenerkennung und die Zahlungsmethoden mit der besten Konversion. Es ist einfach zu integrieren und wird ständig aktualisiert. Checkout ist mit Magento 2 kompatibel.

Das Plugin hält Ihren Magento-Shop und unsere Checkout-Zahlungsplattform synchronisiert, sodass Sie Ihre Bestellungen vollständig in Magento abwickeln und für detailliertere Berichte auf das Checkout Portal zugreifen können.

<Info>
Im Folgenden sehen Sie alle von Checkout angebotenen Zahlungsmethoden. Diese Liste bezieht sich auf alle Märkte.

Je nach Land oder Region kann die Liste variieren. Wenn Sie unsicher sind, ob eine bestimmte Zahlungsmethode in Ihrem Land oder Ihrer Region verfügbar ist, wenden Sie sich bitte an den [Support](https://developer.nexigroup.com/nexi-checkout/en-EU/support/) für weitere Informationen.
</Info>

### Shop-Funktionen

- Der schnelle Checkout mit "Remember Me"-Funktionalität.
- Beinhaltet 2 Checkout-Flows, um Ihrem gewünschten Kundenerlebnis gerecht zu werden: Embedded (Ihre Kunden werden während der Zahlung nicht aus dem Magento-Checkout heraus weitergeleitet) und Hosted Checkout (mit Weiterleitung).
- Eine intelligente Mischung von Zahlungsmethoden für alle Vorlieben.
- Mehrere Checkout-Sprachen und Währungen für den Verkauf im Inland und im Ausland.

![Magento 2 Checkout-Ansicht](https://images.ctfassets.net/di680may5c0k/4nH4jWASY5R0WQFowmnUUV/09b0e266f7d38ed3bb2b941b0837f481/Magento_2_vanilla.png?w=942&h=690&q=50&fm=png)

### Funktionen des Admin-Panels

- Schnelles Setup und flexible Konfiguration.
- Intuitive Bestellverwaltung mit synchronisierten Belastungen und Rückerstattungen.
- Kompatibel mit Rabatten, Steuern (MwSt.) und Versandvarianten.
- Kompatibel mit Multi-Shop-Setups.
- Kompatibel mit mehreren visuellen Themes (Vanilla- und Redirect-Checkouts).

## Installation

Folgen Sie den folgenden Schritten, um das Checkout-Modul in Magento 2 zu installieren und zu aktivieren.

### Schritt 1: Modul installieren

Sie haben zwei Optionen für die Installation des Checkout-Moduls.

#### Alternative 1: Manuell

Laden Sie die Datei manuell auf Ihre Magento-Site hoch:

1. Laden Sie das neueste Checkout-Modul von GitHub herunter: [https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip](https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip)

2: Entpacken Sie die Datei und laden Sie den Inhalt per FTP in das Verzeichnis `/app/code/Dibs/EasyCheckout ` Ihrer Magento-Site hoch.
Bitte erstellen Sie den Ordner `/Dibs/EasyCheckout `, wenn Sie das Plugin zum ersten Mal installieren.

#### Alternative 2: Mit Composer

Eine weitere Option ist die Installation des Moduls über **Composer** auf Ihrer Magento-Site, indem Sie die folgenden Befehle ausführen:

```shell Composer commands
$ composer require nexi-checkout/magento2
```
Es wird dringend empfohlen, die Bereinigung von Pending-Payment-Bestellungen unter Sales > Orders, Orders Cron Settings > Pending Payment Order Lifetime (minutes) zu aktivieren.

### Schritt 2: Modul aktivieren

Aktivieren Sie das Nets Easy-Modul in Magento mit den folgenden Befehlen:

```shell Enable Checkout
$ php bin/magento module:enable --clear-static-content Dibs_EasyCheckout 
$ php bin/magento setup:upgrade
```

Das Modul ist nun installiert und kann für Ihr Checkout-Konto konfiguriert werden.

### Aktualisierung
#### Methode 1: FTP

Wenn Sie das Plugin per FTP installiert haben, können Sie das Modul mit den folgenden Schritten aktualisieren:

1. Laden Sie das neueste Nets Easy-Modul von GitHub herunter: [https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip](https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip)
2. Entfernen Sie den Inhalt des Verzeichnisses /app/code/Dibs/EasyCheckout.
3. Entpacken Sie die heruntergeladene Datei und laden Sie den Inhalt in das Verzeichnis `/app/code/Dibs/EasyCheckout ` hoch.

#### Methode 2: VSC

Wenn Sie das Plugin zuvor durch Hinzufügen des GitHub-Repositorys zu Composer installiert haben, können Sie das Modul mit den folgenden Befehlen aktualisieren:

```shell Update Module
$ composer remove nets/easycheckout
$ composer require nexi-checkout/magento2
$ php bin/magento setup:upgrade
```

#### Methode 3: Composer

Wenn Sie das Plugin über Composer installiert haben, können Sie das Modul mit den folgenden Befehlen aktualisieren:

```shell Update Module
$ composer update nexi-checkout/magento2
$ php bin/magento setup:upgrade
```

## Konfiguration

Nach der Installation des Moduls müssen Sie einige grundlegende Konfigurationen des Moduls vornehmen:

1. Navigieren Sie zu: **System > Configuration > Payment Methods**
2. Suchen Sie **Checkout** und greifen Sie auf **Configuration** zu.
3. Füllen Sie die erforderlichen Felder aus, wie z. B. Merchant ID und Integrationsschlüssel (Secret Keys und Checkout Keys).
3. (Optional) Passen Sie das Modul mit den zusätzlichen Einstellungen auf der Konfigurationsseite an Ihre Bedürfnisse an.

<Info>
Sowohl die **Merchant ID** als auch die **Integrationsschlüssel** finden Sie im [Checkout Portal](https://portal.dibspayment.eu/). Weitere Hilfe finden Sie auf den folgenden Seiten:

- [Wo finde ich meine Händlernummer (Merchant ID)?](https://developer.nexigroup.com/nexi-checkout/en-EU/support/where-can-i-find-my-merchant-number-merchant-id/)
- [Auf Ihre Integrationsschlüssel zugreifen](/access-your-integration-keys)
</Info>

## Bestellverwaltung

Es ist möglich, Bestellungen direkt im Magento-Verwaltungspanel zu verwalten:

1. Navigieren Sie zu **Sales > Order > View**.
2. Klicken Sie auf eine Bestellzeile, um auf die Bestelldetails zuzugreifen.
3. Wählen Sie die gewünschte Aktion, **Rückerstattung/Teil-Rückerstattung**, oder aktualisieren Sie den Bestellstatus. Das Checkout-Plugin synchronisiert automatisch. Der Zahlungsstatus wird auch im [Checkout Portal](https://portal.dibspayment.eu) aktualisiert.

<Info>
Alle von Nexi Group durchgeführten Transaktionen sind im [Checkout Portal](https://portal.dibspayment.eu/) zugänglich.
</Info>

## Fehlerbehebung

Im Folgenden finden Sie einige der häufigsten Konfigurationsfehler, ihre Ursachen und Schritte, die Sie befolgen können, um sie zu beheben.

## Das Nexi Group-Zahlungs-Plugin ist nicht als Zahlungsmethode sichtbar

- Wechseln Sie vorübergehend zum Magento 2-Standardtemplate. Benutzerdefinierte Templates erfordern möglicherweise zusätzliche Änderungen, um eine korrekte Anzeige zu gewährleisten. Wenden Sie sich an Ihren Webdesigner oder Entwickler.

### Das Nexi Group-Zahlungsfenster ist leer

- Stellen Sie sicher, dass Ihre Integrationsschlüssel in den Nexi Group-App-Einstellungen korrekt sind und keine zusätzlichen Leerzeichen enthalten.
- Deaktivieren Sie vorübergehend Plugins von Drittanbietern, die die Funktionalität der App beeinträchtigen könnten.
- Prüfen Sie, ob es vorübergehende technische Probleme gibt: https://easy-status.developers.nets.eu/

### Zahlungen im Live-Modus funktionieren nicht
- Stellen Sie sicher, dass Sie ein genehmigtes Live Checkout-Konto für die Produktion haben.
- Stellen Sie sicher, dass Ihr Live Checkout-Konto für Zahlungen mit der gewählten Währung freigegeben ist.
- Stellen Sie sicher, dass die Zahlungsmethodendaten korrekt sind und von Ihrem Checkout-Vertrag unterstützt werden.

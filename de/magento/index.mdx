## Übersicht

Unser Plugin für Magento 2 unterstützt alle Checkout-Funktionen wie die intelligente Kundenerkennung und die am besten konvertierenden Zahlungsmethoden. Es ist einfach zu integrieren und wird ständig aktualisiert. Checkout ist mit Magento 2 kompatibel.

Das Plugin hält Ihren Magento-Shop und unsere Checkout-Zahlungsplattform synchronisiert, sodass Sie Ihre Bestellungen vollständig in Magento bearbeiten können und für detailliertere Berichte auf das Checkout Portal zugreifen können.

<Info>
Nachfolgend sehen Sie alle von Checkout angebotenen Zahlungsmethoden. Diese Liste bezieht sich auf alle Märkte.

Je nach Land oder Region kann die Liste variieren. Wenn Sie bei einer bestimmten Zahlungsmethode unsicher sind, ob diese in Ihrem Land oder Ihrer Region verfügbar ist, kontaktieren Sie bitte den [Support](https://developer.nexigroup.com/nexi-checkout/en-EU/support/) für weitere Informationen.
</Info>

### Shop-Funktionen

- Der schnelle Checkout mit "Remember me"-Funktion.
- Beinhaltet 2 Checkout-Abläufe, um dem gewünschten Kundenerlebnis zu entsprechen: Embedded (Ihre Kunden werden zur Bezahlung nicht aus dem Magento-Checkout umgeleitet) und Hosted Checkout (mit Weiterleitung).
- Eine intelligente Mischung von Zahlungsmethoden für alle Vorlieben.
- Mehrere Checkout-Sprachen und Währungen für den Verkauf im In- und Ausland.

![Magento 2 Checkout-Ansicht](https://images.ctfassets.net/di680may5c0k/4nH4jWASY5R0WQFowmnUUV/09b0e266f7d38ed3bb2b941b0837f481/Magento_2_vanilla.png?w=942&h=690&q=50&fm=png)

### Funktionen des Admin-Panels

- Schnelle Einrichtung und flexible Konfiguration.
- Intuitive Auftragsverwaltung mit synchronisierten Charges und Refunds.
- Kompatibel mit Rabatten, Steuer (MwSt.) und Versandvarianten.
- Kompatibel mit Multi-Shop-Setups.
- Kompatibel mit mehreren visuellen Themes (Vanilla- und Redirect-Checkouts).

## Installation

Folgen Sie den nachstehenden Schritten, um das Checkout-Modul in Magento 2 zu installieren und zu aktivieren.

### Schritt 1: Modul installieren

Sie haben zwei Möglichkeiten zur Installation des Checkout-Moduls.

#### Alternative 1: Manuell

Laden Sie die Datei manuell auf Ihre Magento-Seite hoch:

1. Laden Sie das aktuellste Checkout-Modul von GitHub herunter: [https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip](https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip)

2: Entpacken Sie die Datei und laden Sie den Inhalt per FTP in das Verzeichnis `/app/code/Dibs/EasyCheckout ` Ihrer Magento-Seite hoch.
Bitte legen Sie den Ordner `/Dibs/EasyCheckout ` beim ersten Installieren des Plugins an.

#### Alternative 2: Mit Composer

Eine weitere Option ist die Installation des Moduls über **Composer** auf Ihrer Magento-Seite, indem Sie folgende Befehle ausführen:

```shell Composer commands
$ composer require nexi-checkout/magento2
```
Es wird dringend empfohlen, die Bereinigung von ausstehenden Zahlungsbestellungen unter Sales > Orders, Orders Cron Settings > Pending Payment Order Lifetime (minutes) zu aktivieren.

### Schritt 2: Modul aktivieren

Aktivieren Sie das Nets Easy-Modul in Magento, indem Sie folgende Befehle ausführen:

```shell Enable Checkout
$ php bin/magento module:enable --clear-static-content Dibs_EasyCheckout 
$ php bin/magento setup:upgrade
```

Das Modul ist nun installiert und kann für Ihr Checkout-Konto konfiguriert werden.

### Aktualisieren
#### Methode 1: FTP

Wenn Sie das Plugin per FTP installiert haben, können Sie das Modul wie folgt aktualisieren:

1. Laden Sie das aktuellste Nets Easy-Modul von GitHub herunter: [https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip](https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip)
2. Entfernen Sie den Inhalt des Verzeichnisses /app/code/Dibs/EasyCheckout.
3. Entpacken Sie die heruntergeladene Datei und laden Sie den Inhalt in das Verzeichnis `/app/code/Dibs/EasyCheckout ` hoch.

#### Methode 2: VSC

Wenn Sie das Plugin zuvor installiert haben, indem Sie das GitHub-Repository zu Composer hinzugefügt haben, können Sie das Modul mit folgenden Befehlen aktualisieren:

```shell Update Module
$ composer remove nets/easycheckout
$ composer require nexi-checkout/magento2
$ php bin/magento setup:upgrade
```

#### Methode 3: Composer

Wenn Sie das Plugin über Composer installiert haben, können Sie das Modul mit folgenden Befehlen aktualisieren:

```shell Update Module
$ composer update nexi-checkout/magento2
$ php bin/magento setup:upgrade
```

## Konfiguration

Nach der Installation des Moduls müssen Sie einige grundlegende Konfigurationen vornehmen:

1. Navigieren Sie zu: **System > Konfiguration > Zahlungsmethoden**
2. Suchen Sie **Checkout** und gehen Sie zur **Konfiguration**.
3. Füllen Sie die erforderlichen Felder aus, wie z. B. Händler-ID und Integrationsschlüssel (Secret-Keys und Checkout-Keys).
3. (Optional) Passen Sie das Modul mit den zusätzlichen Einstellungen auf der Konfigurationsseite Ihren Bedürfnissen an.

<Info>
Sowohl die **Händler-ID** als auch die **Integrationsschlüssel** finden Sie im [Checkout Portal](https://portal.dibspayment.eu/). Weitere Informationen finden Sie auf den folgenden Seiten:

- [Wo finde ich meine Händlernummer (Merchant-ID)?](https://developer.nexigroup.com/nexi-checkout/en-EU/support/where-can-i-find-my-merchant-number-merchant-id/)
- [Auf Ihre Integrationsschlüssel zugreifen](/access-your-integration-keys)
</Info>

## Auftragsverwaltung

Es ist möglich, Bestellungen direkt im Magento-Administrationsbereich zu verwalten:

1. Navigieren Sie zu **Sales > Order > View**.
2. Klicken Sie auf eine Bestellzeile, um die Bestelldetails aufzurufen.
3. Wählen Sie die gewünschte Aktion, **Erstattung/Teilerstattung**, oder aktualisieren Sie den Bestellstatus. Das Checkout-Plugin synchronisiert sich automatisch. Der Zahlungsstatus wird auch im [Checkout Portal](https://portal.dibspayment.eu) aktualisiert.

<Info>
Alle von der Nexi Group durchgeführten Transaktionen sind im [Checkout Portal](https://portal.dibspayment.eu/) zugänglich.
</Info>

## Fehlerbehebung

Nachfolgend finden Sie einige der häufigsten Konfigurationsfehler, deren Ursachen und Schritte zu deren Behebung.

## Nexi Group Zahlungs-Plugin ist nicht als Zahlungsmethode sichtbar

- Wechseln Sie vorübergehend zum Standard-Template von Magento 2. Benutzerdefinierte Templates erfordern möglicherweise zusätzliche Anpassungen, um eine korrekte Anzeige sicherzustellen. Wenden Sie sich an Ihren Webdesigner oder Entwickler.

### Nexi Group Zahlungsfenster ist leer

- Stellen Sie sicher, dass Ihre Integrationsschlüssel in den Nexi Group App-Einstellungen korrekt sind und keine zusätzlichen Leerzeichen enthalten.
- Deaktivieren Sie vorübergehend Drittanbieter-Plugins, die die Funktionalität der App beeinträchtigen könnten.
- Prüfen Sie, ob es vorübergehende technische Inkonsistenzen gibt: https://easy-status.developers.nets.eu/

### Zahlungen im Live-Modus funktionieren nicht
- Stellen Sie sicher, dass Sie über ein genehmigtes Live-Checkout-Konto für die Produktion verfügen.
- Stellen Sie sicher, dass Ihr Live-Checkout-Konto für Zahlungen in der gewählten Währung freigegeben ist.
- Stellen Sie sicher, dass die Zahlungsmethodendaten korrekt sind und durch Ihren Checkout-Vertrag unterstützt werden.

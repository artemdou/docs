## Überblick

Unser Plugin für Magento 2 unterstützt alle Checkout-Funktionen wie die intelligente Kundenerkennung und die am besten konvertierenden Zahlungsmethoden. Es lässt sich einfach integrieren und wird ständig aktualisiert. Checkout ist mit Magento 2 kompatibel.

Das Plugin hält Ihren Magento-Shop und unsere Checkout-Zahlungsplattform synchron, sodass Sie Ihre Bestellungen vollständig in Magento abwickeln und für detailliertere Berichte das Checkout-Portal nutzen können.

<Info>
Nachfolgend sehen Sie alle von Checkout angebotenen Zahlungsmethoden. Diese Liste bezieht sich auf alle Märkte.

Je nach Land oder Region kann die Liste variieren. Wenn Sie sich bei einer bestimmten Zahlungsmethode unsicher sind und ob diese in Ihrem Land oder Ihrer Region verfügbar ist, wenden Sie sich bitte an den [Support](https://developer.nexigroup.com/nexi-checkout/en-EU/support/) für weitere Informationen.
</Info>

### Shop-Funktionen

- Das schnelle Checkout mit „Remember me"-Funktionalität.
- Beinhaltet 2 Checkout-Abläufe, um Ihrer gewünschten Kundenerfahrung zu entsprechen: eingebettet (Ihre Kunden werden zur Zahlung nicht außerhalb des Magento-Checkouts weitergeleitet) und gehostetes Checkout (mit Weiterleitung).
- Ein intelligenter Mix aus Zahlungsmethoden, der allen Vorlieben gerecht wird.
- Mehrere Checkout-Sprachen und -Währungen für den Verkauf im In- und Ausland.

![Magento 2 Checkout-Ansicht](https://images.ctfassets.net/di680may5c0k/4nH4jWASY5R0WQFowmnUUV/09b0e266f7d38ed3bb2b941b0837f481/Magento_2_vanilla.png?w=942&h=690&q=50&fm=png)

- Schnelle Einrichtung und flexible Konfiguration.

- Kompatibel mit Rabatten, Steuern (MwSt.) und Versandvarianten.
- Kompatibel mit Multi-Shop-Setups.
- Kompatibel mit mehreren visuellen Themes (Vanilla- und Redirect-Checkouts).

## Installation

Folgen Sie den nachstehenden Schritten, um das Checkout-Modul in Magento 2 zu installieren und zu aktivieren.

### Schritt 1: Modul installieren

Sie haben zwei Möglichkeiten, das Checkout-Modul zu installieren.

#### Alternative 1: Manuell

Laden Sie die Datei manuell auf Ihre Magento-Website hoch:

1. Laden Sie das neueste Checkout-Modul von GitHub herunter: [https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip](https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip)

2: Entpacken Sie die Datei und laden Sie den Inhalt per FTP in das Verzeichnis `/app/code/Dibs/EasyCheckout ` Ihrer Magento-Website hoch.
Bitte erstellen Sie den Ordner `/Dibs/EasyCheckout `, wenn Sie das Plugin zum ersten Mal installieren.

#### Alternative 2: Mit Composer

Eine andere Möglichkeit ist die Installation des Moduls per **Composer** auf Ihrer Magento-Website mit folgenden Befehlen:

```shell Composer commands
$ composer require nexi-checkout/magento2
```
Es wird dringend empfohlen, die Bereinigung von Bestellungen mit Status „Pending Payment" zu aktivieren unter Sales > Orders, Orders Cron Settings > Pending Payment Order Lifetime (minutes).

### Schritt 2: Modul aktivieren

Aktivieren Sie das Nets-Easy-Modul in Magento mit den folgenden Befehlen:

```shell Enable Checkout
$ php bin/magento module:enable --clear-static-content Dibs_EasyCheckout 
$ php bin/magento setup:upgrade
```

Das Modul ist nun installiert und bereit, für Ihr Checkout-Konto konfiguriert zu werden.

### Aktualisierung
#### Methode 1: FTP

Wenn Sie das Plugin per FTP installiert haben, können Sie das Modul anhand der folgenden Schritte aktualisieren:

1. Laden Sie das neueste Nets-Easy-Modul von GitHub herunter: [https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip](https://github.com/Nets-eCom/Magento2_easy/archive/refs/heads/master.zip)
2. Entfernen Sie den Inhalt des Verzeichnisses /app/code/Dibs/EasyCheckout.
3. Entpacken Sie die heruntergeladene Datei und laden Sie den Inhalt in das Verzeichnis `/app/code/Dibs/EasyCheckout ` hoch.

#### Methode 2: VSC

Wenn Sie das Plugin zuvor installiert haben, indem Sie das GitHub-Repository zu Composer hinzugefügt haben, können Sie das Modul mit folgenden Befehlen aktualisieren:

```shell Update Module
$ composer remove nets/easycheckout
$ composer require nexi-checkout/magento2
$ php bin/magento setup:upgrade
```

#### Methode 3: Composer

Wenn Sie das Plugin über Composer installiert haben, können Sie das Modul mit folgenden Befehlen aktualisieren:

```shell Update Module
$ composer update nexi-checkout/magento2
$ php bin/magento setup:upgrade
```

## Konfiguration

Nach der Installation des Moduls müssen Sie einige grundlegende Konfigurationen vornehmen:

1. Navigieren Sie zu: **System > Configuration > Payment Methods**
2. Suchen Sie **Checkout** und gehen Sie zu **Configuration**.
3. Füllen Sie die erforderlichen Felder aus, wie z. B. Händler-ID und Integrationsschlüssel (geheime Schlüssel und Checkout-Schlüssel).
3. (Optional) Passen Sie das Modul mithilfe der zusätzlichen Einstellungen auf der Konfigurationsseite an Ihre Bedürfnisse an.

<Info>
Sowohl die **Händler-ID** als auch die **Integrationsschlüssel** finden Sie im [Checkout-Portal](https://portal.dibspayment.eu/). Weitere Hilfe finden Sie auf den folgenden Seiten:

- [Wo finde ich meine Händlernummer (Händler-ID)?](https://developer.nexigroup.com/nexi-checkout/en-EU/support/where-can-i-find-my-merchant-number-merchant-id/)
- [Auf Ihre Integrationsschlüssel zugreifen](/access-your-integration-keys)
</Info>

## Auftragsverwaltung

Es ist möglich, Bestellungen direkt im Magento-Verwaltungsbereich zu verwalten:

1. Navigieren Sie zu **Sales > Order > View**.
2. Klicken Sie auf eine Bestellzeile, um auf die Bestelldetails zuzugreifen.
3. Wählen Sie die gewünschte Aktion, **Rückerstattung/Teilrückerstattung**, oder aktualisieren Sie den Bestellstatus. Das Checkout-Plugin synchronisiert automatisch. Der Zahlungsstatus wird auch im [Checkout-Portal](https://portal.dibspayment.eu) aktualisiert.

<Info>
Alle von Nexi Group durchgeführten Transaktionen sind im [Checkout-Portal](https://portal.dibspayment.eu/) abrufbar.
</Info>

## Fehlerbehebung

Nachfolgend finden Sie einige der häufigsten Konfigurationsfehler, ihre Ursache und Schritte, mit denen Sie sie beheben können.

## Nexi Group-Zahlungs-Plugin ist nicht als Zahlungsmethode sichtbar

- Wechseln Sie vorübergehend zur Magento-2-Standardvorlage. Benutzerdefinierte Vorlagen erfordern möglicherweise zusätzliche Anpassungen, um eine korrekte Anzeige zu gewährleisten. Wenden Sie sich an Ihren Webdesigner oder Entwickler.

### Nexi Group-Zahlungsfenster ist leer

- Stellen Sie sicher, dass Ihre Integrationsschlüssel in den Nexi Group-App-Einstellungen korrekt sind und keine zusätzlichen Leerzeichen enthalten.
- Deaktivieren Sie vorübergehend Drittanbieter-Plugins, die die Funktionalität der App beeinträchtigen könnten.
- Prüfen Sie, ob es vorübergehende technische Probleme gibt: https://easy-status.developers.nets.eu/

### Zahlungen im Live-Modus funktionieren nicht
- Stellen Sie sicher, dass Sie ein genehmigtes Live-Checkout-Konto für die Produktion haben.
- Stellen Sie sicher, dass Ihr Live-Checkout-Konto für Zahlungen mit der ausgewählten Währung freigegeben ist.
- Stellen Sie sicher, dass die Zahlungsmethodendaten korrekt sind und von Ihrem Checkout-Vertrag unterstützt werden.

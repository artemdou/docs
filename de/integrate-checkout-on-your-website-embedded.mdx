## Bevor Sie beginnen

<Warning>
Bevor Sie beginnen, benötigen Sie:

1. Ein [Checkout Portal-Konto](https://portal.dibspayment.eu). Weitere Informationen finden Sie unter [Konto erstellen](/create-a-checkout-portal-account).
2. Ihre [Integrationsschlüssel](https://portal.dibspayment.eu/integration) für die Website, die Sie entwickeln. [Brauchen Sie Hilfe](/access-your-integration-keys)?
3. Einen laufenden Webserver, der HTML-Seiten hosten und serverseitige Skripte ausführen kann.
</Warning>

## Was Sie erstellen

Diese Anleitung zeigt Ihnen, wie Sie einen **eingebetteten Checkout** in Ihre Website integrieren. Das folgende Bild zeigt die Beziehungen zwischen dem Kunden, Ihrer Website und Checkout.

![Eingebetteter Checkout](https://images.ctfassets.net/di680may5c0k/4fPy5dg1J0AG1owr1CLCd0/d2f755746cd1354b501dab2a190ec24c/Direct_integration_embedded.png?w=3060&h=1536&q=50&fm=png)

Der **Kunde** interagiert bei einem eingebetteten Checkout ausschließlich mit *Ihrer* Website. Das **Backend** Ihrer Website kommuniziert mit der Nexi Group über die RESTful [Payment API](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1) unter Verwendung Ihres [Secret API Keys](/access-your-integration-keys). Das **Frontend** Ihrer Website verwendet das von der Nexi Group bereitgestellte [Checkout JS SDK](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/checkout-js-sdk), um die API-Integration mit Checkout zu handhaben. Das Frontend Ihrer Website verwendet den [Checkout Key](/access-your-integration-keys), um Ihren Webshop bei der Kommunikation mit der Nexi Group zu identifizieren.

<Warning>
Der Secret API Key darf niemals der Öffentlichkeit zugänglich gemacht werden und sollte nur über verschlüsselte Server-zu-Server-Kommunikation (HTTPS) übertragen werden.
Im Gegensatz dazu ist der Checkout Key nur eine Kennung und kann dem Frontend Ihrer Website zur Verfügung gestellt werden.
</Warning>

<Steps>
  <Step title="Den Checkout von Ihrer Website aus initiieren (Frontend)">
    Der Checkout wird vom Client initiiert. Um die Implementierung des Checkout-Ablaufs vom Frontend zu starten, müssen Sie Folgendes hinzufügen:

    - Eine Schaltfläche, mit der der Kunde den Checkout initiieren kann
    - Einen JavaScript-Event-Handler, der an die Schaltfläche angehängt ist

    Der minimale Warenkorb unten dient als Ausgangspunkt für den Zahlungsablauf:

    ### Warenkorb

    <CodeGroup>

    ```html cart.html
    <!DOCTYPE html>
    <html>
      <head>
        <title>Shopping Cart</title>
      </head>
      <body>
        <h1>Shopping Cart</h1>
        <button id="checkout-button">Proceed to Checkout</button>
        <script type="text/javascript">
        var button = document.getElementById('checkout-button');
        button.addEventListener('click', function () {
          var request = new XMLHttpRequest();

          // create-payment.php is implemented in Step 2
          request.open('GET', 'create-payment.php', true); 
          request.onload = function () {
            const data = JSON.parse(this.response);        // If parse error, check output 
            if (!data.paymentId) {                         // from create-payment.php
              console.error('Error: Check output from create-payment.php');
              return;
            }
            console.log(this.response);

            // checkout.html is implemented in Step 3
            window.location = 'checkout.html?paymentId=' + data.paymentId;
          }
          request.onerror = function () { console.error('connection error'); }
          request.send();
        });
       </script>
      </body>
    </html>
    ```

    ```html cart.html
    <!DOCTYPE html>
    <html>
      <head>
        <title>Shopping Cart</title>
      </head>
      <body>
        <h1>Shopping Cart</h1>
        <button id="checkout-button">Proceed to Checkout</button>
        <script type="text/javascript">
        var button = document.getElementById('checkout-button');
        button.addEventListener('click', function () {

          // The node server that responds to the 
          // create-payment request is implemented in Step 2
          request.open('GET', 'create-payment', true);

       </script>
    ```

    </CodeGroup>

    Diese Anleitung verwendet [Express](https://expressjs.com), um statische Dateien (HTML und Frontend-JS) zu hosten. Das folgende Skript dient als Ausgangspunkt und wird in den nächsten Schritten erweitert. Aktuell werden statische HTML-Seiten aus dem Verzeichnis `public ` ausgeliefert:

    ```javascript server.js
    const http = require("https");
    const express = require('express')
    const app = express()
    const port = 8080
    const fs = require('fs');

    // Host static html pages and JavaScripts
    // from the "public" subdirectory
    app.use(express.static('public'))

    app.listen(port, () => {
            console.log(`Server listening on port ${port}`)
    })
    ```

    <Info>
    **Fehlerbehebung**

    - Legen Sie die Datei `cart.html ` in einem Unterverzeichnis namens `public ` ab.
    - Fügen Sie der URL den Port (8080) hinzu, wenn Sie den Warenkorb in Ihrem Browser laden.
    </Info>

    Wenn Sie auf die Checkout-Schaltfläche klicken, sendet der Event-Handler eine asynchrone Anfrage über HTTP(S) an das Backend Ihrer Website. Wenn Sie jetzt auf die Checkout-Schaltfläche klicken, erhalten Sie einen Fehler, da Sie das Backend noch nicht implementiert haben. Lassen Sie uns mit dem nächsten Schritt fortfahren und diesen Fehler beheben.
  </Step>
  <Step title="Ein Zahlungsobjekt erstellen (Backend)">
    Jede **Zahlungssitzung** wird durch ein **Zahlungsobjekt** dargestellt. Um einen Checkout-Ablauf für Ihren Kunden zu starten, müssen Sie zunächst [ein Zahlungsobjekt erstellen](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post) und die `paymentId ` abrufen, die auf dieses Objekt verweist. Die Erstellung eines Zahlungsobjekts erfordert Ihren [Secret API Key](/access-your-integration-keys). Daher muss diese Anfrage vom Backend Ihrer Website initiiert werden.

    Das folgende Backend-Skript erstellt ein neues Zahlungsobjekt und gibt die `paymentId ` an das Frontend zurück:

    ### Zahlung erstellen

    <CodeGroup>

    ```php create-payment.php
    <?php

    $payload = file_get_contents('payload.json');
    assert(json_decode($payload) && json_last_error() == JSON_ERROR_NONE);

    $ch = curl_init('https://test.api.dibspayment.eu/v1/payments');
    curl_setopt($ch, CURLOPT_CUSTOMREQUEST, 'POST');
    curl_setopt($ch, CURLOPT_POSTFIELDS, $payload);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
    curl_setopt($ch, CURLOPT_HTTPHEADER, array(                                                                         
            'Content-Type: application/json',
            'Accept: application/json',
            'Authorization: <YOUR_SECRET_API_KEY>'));                                                
    $result = curl_exec($ch);
    echo($result);
    ```

    ```javascript server.js
    const http = require("https");
    const express = require('express')
    const app = express()
    const port = 8080
    const fs = require('fs');

    // Host static html pages and JavaScripts
    // from the "public" subdirectory
    app.use(express.static('public'))

    app.get('/create-payment', (req, res) => {

    	const options = {
    		"method": "POST",
    		"hostname": "test.api.dibspayment.eu",
    		"port": 443,
    		"path": "/v1/payments",
    		"headers": {
    			"content-type": "application/json",
    			"Authorization": "<YOUR_SECRET_API_KEY>"
    		}
    	};

    	const restreq = http.request(options, function (resp) {
    		const chunks = [];

    		console.log("statusCode: ", resp.statusCode);
    		console.log("headers: ", resp.headers);

    		resp.on("data", function (chunk) {
    			console.log("on data");
    			chunks.push(chunk);
    		});
    		resp.on("end", function () {
    			const body = Buffer.concat(chunks);
    			console.log(body.toString());
    			res.send(body.toString());
    		});
    	});

      // Read hard-coded payload from file in this example.
    	// This is where you would normally generate a 
    	// json object dynamically based on the current order.
    	let payload = fs.readFileSync('payload.json');
    	restreq.write(payload);
    	// console.log(JSON.parse(payload));

    	restreq.on('error', function (e) {
    		console.error('error');
    		console.error(e);

    	});
    	restreq.end();
    })

    app.listen(port, () => {
    	console.log(`Server listening on port ${port}`)
    })
    ```

    </CodeGroup>

    Ersetzen Sie den Text `&lt;YOUR_SECRET_API_KEY&gt;` durch Ihren [Secret API Key](https://portal.dibspayment.eu/integration).

    In diesem Beispiel lesen wir einfach ein hartcodiertes JSON-Objekt aus der unten gezeigten Datei `payload.json `. Normalerweise würden Sie an dieser Stelle dynamisch ein JSON-Objekt basierend auf den Bestellpositionen Ihres Kunden erstellen. Vorerst können Sie einfach die statische JSON-Datei `payload.json ` hinzufügen, die zum Testen verwendet werden kann:

    ```json payload.json
    {
      "checkout": {
        "integrationType": "EmbeddedCheckout",
        "url": "https://<YOUR_SERVER>/checkout.html",
        "termsUrl": "https://<YOUR_SERVER>/terms.html"
      },
      "order": {
        "items": [
          {
            "reference": "ref42",
            "name": "Demo product",
            "quantity": 2,
            "unit": "pcs",
            "unitPrice": 80000,
            "grossTotalAmount": 160000,
            "netTotalAmount": 160000
          },
            {
            "reference": "discount",
            "name": "Demo discount",
            "quantity": 1,
            "unit": "pcs",
            "unitPrice": -20000,
            "grossTotalAmount": -20000,
            "netTotalAmount": -20000
            }
        ],
        "amount": 140000,
        "currency": "SEK",
        "reference": "Demo Order"
      }
    }
    ```

    Der Anforderungsbody enthält:

    - **Integrationstyp**: Entweder ein eingebetteter Checkout, der einen `iframe ` auf Ihrer Seite verwendet, oder eine vorgefertigte, von der Nexi Group gehostete Checkout-Seite, zu der der Kunde weitergeleitet wird.
    - **Checkout-URL**: Die URL zu Ihrer Checkout-Seite. Diese URL muss exakt mit der URL Ihrer Checkout-Seite übereinstimmen, einschließlich Protokoll (http/https) und vollqualifiziertem Domainnamen (FQDN).
    - **Terms-URL**: Eine URL zu einer Seite auf Ihrer Website, die die Zahlungsbedingungen Ihres Webshops beschreibt.
    - **Bestelldetails**: Enthält Bestellpositionen, Gesamtbetrag und Währung

    > info
    >
    > Stellen Sie sicher, dass Sie die folgende Zeile ersetzen:
    >
    > `"url": "https://&lt;YOUR_SERVER&gt;/checkout.html"`
    >
    > durch die **exakte URL** zur Checkout-Seite (im nächsten Schritt erstellt),
    > sonst kann die Seite später nicht geladen werden. Überprüfen Sie das **Protokoll** (http/https), die **Serveradresse** und den **Port**, den Sie verwenden.

    Dies ist ein einfaches Beispiel, um Ihnen den Einstieg zu erleichtern. Beim Erstellen oder Aktualisieren eines Zahlungsobjekts gibt es zahlreiche Einstellungen, die hier nicht behandelt werden. Weitere Informationen finden Sie in der Anleitung [Versandkosten hinzufügen](/add-shipping-cost) und in der [Payment API-Referenz](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#payments).

    Sie sollten nun in der Lage sein, auf die Checkout-Schaltfläche zu klicken und die `paymentId ` in der JavaScript-Konsole Ihres Webbrowsers ausgegeben zu sehen.

    <Info>
    **Fehlerbehebung**

    Wenn die `paymentId ` nicht in der JavaScript-Konsole ausgegeben wird, können Sie auch versuchen, die URL zum Backend-Skript (Create Payment) direkt in Ihrem Webbrowser zu laden.
    Auf diese Weise können Sie die vom Skript ausgegebenen Fehlermeldungen leichter lesen.
    </Info>

    Nach der Implementierung des Backends ist es Zeit, zum Frontend-Code zurückzukehren und die `paymentId ` zu verwenden, um die Checkout-Seite mit der Zahlungsansicht zu erstellen.
  </Step>
  <Step title="Eine Checkout-Seite hinzufügen (Frontend)">
    Jetzt ist es Zeit, die HTML-Seite zu erstellen, die den Checkout-`iframe ` einbetten wird. Fügen Sie den folgenden HTML-Code in eine neue Datei ein:

    ```html checkout.html
    <!DOCTYPE html>
    <html>
     <head><title>Checkout</title></head>
     <body>
      <h1>Checkout</h1>
       <div id="checkout-container-div">
         
       </div>
       <script src="https://test.checkout.dibspayment.eu/v1/checkout.js?v=1"></script>
       <script src="script.js"></script>
     </body>
    </html>
    ```

    Beachten Sie einige Dinge zum HTML-Code:

    - Das Container-Element `<div id="checkout-container-div">` ist die Stelle, an der der Checkout-`iframe ` schließlich eingebettet wird.
    - Es werden zwei JavaScripts geladen: `checkout.js ` von **Nexi Group** und das benutzerdefinierte `script.js `, das Sie im nächsten Schritt implementieren werden.
    - Da diese Anleitung die [Easy-Testumgebung](/test-environment) verwendet, wird `checkout.js ` von `test.checkout.dibspayment.eu ` geladen. In der Live-Umgebung sollten Sie stattdessen `checkout.dibspayment.eu ` verwenden.

    Die Seite `checkout.html ` sollte immer mit einem URL-Parameter namens `paymentId ` aufgerufen werden. Dies ist erforderlich, da die `paymentId ` benötigt wird, um die aktuelle Zahlungssitzung bei der Kommunikation mit der Nexi Group zu identifizieren.

    Wenn die `paymentId ` ausschließlich in der Sitzung gespeichert wird, kann sie bei der Weiterleitung zur und von der Checkout-URL für bestimmte Zahlungsmethoden bei Verwendung unseres eingebetteten Checkouts verloren gehen.

    Um dies zu verhindern, fügen wir den Query-Parameter beim Zurückkehren von einem Drittanbieter zur Checkout-URL hinzu. Dies erfordert, dass der Checkout nach dem Query-Parameter `paymentId ` sucht und in der Lage ist, den Checkout mit dem angegebenen Wert zu initialisieren.
  </Step>
  <Step title="Checkout-Iframe mit dem Checkout JS SDK erzeugen">
    Jetzt ist es Zeit, das JavaScript zu erstellen, das mit Checkout über das [Checkout JS SDK](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/checkout-js-sdk/) kommuniziert.

    Erstellen Sie eine neue Datei und fügen Sie den folgenden JavaScript-Code hinzu:

    ```javascript script.js

    document.addEventListener('DOMContentLoaded', function () {
      const urlParams = new URLSearchParams(window.location.search);
      const paymentId = urlParams.get('paymentId');
      if (paymentId) {
        const checkoutOptions = {
          checkoutKey: '<YOUR_CHECKOUT_KEY>', // Replace!
          paymentId: paymentId,
          containerId: "checkout-container-div",
        };
        const checkout = new Dibs.Checkout(checkoutOptions);
        checkout.on('payment-completed', function (response) {
          window.location = 'completed.html';
        });
      } else {
        console.log("Expected a paymentId");   // No paymentId provided, 
        window.location = 'cart.html';         // go back to cart.html
      }
    });
    ```

    Ersetzen Sie `&lt;YOUR_CHECKOUT_KEY&gt;` durch Ihren [Checkout Key](https://portal.dibspayment.eu/integration) für die [Testumgebung](/test-environment).

    Damit das `checkout.js `-Skript korrekt geladen wird, ist es wichtig, dass Sie die richtige URL zu Ihrer Checkout-Seite in [Schritt 3](#build-step-3-add-a-checkout-page-frontend) angegeben haben.

    Das Checkout-Objekt löst das Ereignis `'payment-completed'` aus, sobald die Zahlung abgeschlossen ist. Die Callback-Funktion, die wir bereitstellen, navigiert zu `completed.html `, das Sie im nächsten Schritt erstellen werden.

    <Info>
    **Fehlerbehebung**

    - Stellen Sie sicher, dass Sie die richtigen [Integrationsschlüssel](https://portal.dibspayment.eu/integration) für die [Testumgebung](/test-environment) verwenden.
    - Damit das `checkout.js `-Skript korrekt geladen wird, stellen Sie sicher, dass Sie die richtige URL zu Ihrer Checkout-Seite angegeben haben. Siehe [Schritt 3](#step-3-add-a-checkout-page-frontend).
    </Info>
  </Step>
  <Step title="Eine 'Bestellung erfolgreich'-Seite hinzufügen">
    Fügen Sie den folgenden Code zu Ihrer Website hinzu:

    ```html completed.html
    <!DOCTYPE html>
    <html>
      <head>
        <title>Payment completed</title>
      </head>
      <body>
        <h1>Payment completed!</h1>
        <a href="cart.html">Back to cart</a>
      </body>
    </html>
    ```

    Das war's! Es ist Zeit zu testen und zu überprüfen, ob Ihre neue eingebettete Checkout-Seite funktioniert.
  </Step>
  <Step title="Testen Sie Ihre Checkout-Seite">
    Sie sollten nun eine rudimentäre Checkout-Seite haben, die mit den auf der Seite [Test der Kartenverarbeitung](/test-card-processing) verfügbaren Testkarten getestet werden kann.

    So testen Sie Ihre neue Checkout-Seite:

    1. Laden Sie die Warenkorbseite neu und klicken Sie auf die Schaltfläche "Proceed to Checkout".
    3. Füllen Sie das Adressformular mit Ihrer **E-Mail-Adresse**, **Telefonnummer** und **Postleitzahl** aus.
    4. Geben Sie eine der [Beispiel-Kartennummern](/test-card-processing#build-sample-credit-cards) ein und klicken Sie auf die Schaltfläche "Pay".
    5. Probieren Sie es aus!

    Die simulierte Zahlungsverarbeitung sollte nun starten und Sie schließlich zurück zur "Payment completed"-Seite führen, die Sie im vorherigen Schritt erstellt haben.

    <Info>
    **Fehlerbehebung**

    - Überprüfen Sie, ob Sie die richtigen [Integrationsschlüssel](/access-your-integration-keys) für die [Testumgebung](/test-environment) verwenden.
    - `checkout.js ` kann nicht geladen werden, sofern Sie nicht die richtige Checkout-URL im JSON-Body angeben, der in [Schritt 2](#build-step-2-create-a-payment-object-backend) beim Erstellen der `paymentId ` angegeben wurde.
    - Wenn Sie die Checkout-URL ändern, laden Sie die Warenkorbseite neu, damit eine neue Zahlungssitzung erstellt wird (und eine neue `paymentId `).
    </Info>

    > tada
    > **Herzlichen Glückwunsch!**
    >
    > Sie haben nun einen eingebetteten Checkout integriert.
  </Step>
</Steps>

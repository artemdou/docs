## Checkout-Übersicht – Versandkosten

Wenn Sie Waren an Ihre Kunden versenden, möchten Sie wahrscheinlich Versandkosten zu deren Bestellung hinzufügen. Sind die Versandkosten ein fester Betrag unabhängig von der Anzahl der Warenkorbartikel oder der Lieferadresse, können Sie einfach einen Artikel zur Bestellung hinzufügen, der die Versandkosten repräsentiert, wenn Sie das Zahlungsobjekt erstmals erstellen.

Allerdings variieren die Versandkosten häufig je nach Anzahl der Artikel im Warenkorb und der Lieferadresse. In diesem Fall können die Versandkosten erst berechnet werden, wenn Ihr Kunde eine Lieferadresse für die Bestellung angegeben hat.

Checkout ermöglicht Ihnen die Verwaltung der Versandkosten auf der Checkout-Seite. Dies erfordert, dass sowohl das Frontend als auch das Backend mit Checkout interagieren. Das folgende Flussdiagramm zeigt den Prozess zur Aktualisierung der Versandkosten basierend auf dem von Ihrem Kunden angegebenen Bestimmungsort.

![Versandkosten](https://images.ctfassets.net/di680may5c0k/5TKYQ9UJOpfU5vaTIsgVme/2ca2ac4e38279dc3802b27faa3506131/Shipping_cost.png?w=2704&h=1726&q=50&fm=png)

Diese Anleitung zeigt, wie Sie diese Funktionalität bei einer eingebetteten Web-Integration implementieren. Die eigentliche Berechnung der Versandkosten müssen Sie auf Ihrer Seite implementieren; sie wird in dieser Anleitung nicht bereitgestellt.

<Steps>
  <Step title="Zahlungsobjekt erstellen (Backend)">
    Checkout bietet Ihnen die Möglichkeit, den Checkout-Ablauf anzuhalten, falls keine Versandkosten angegeben sind. Um diese Funktion zu aktivieren, setzen Sie die Eigenschaft [`merchantHandlesShippingCost `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post-body-checkout-shipping-merchanthandlesshippingcost) beim [Erstellen des Zahlungsobjekts](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post) auf `true `.

    Das folgende Snippet zeigt, wie Sie die Eigenschaft `merchantHandlesShippingCost ` beim Erstellen des Zahlungsobjekts angeben:

    ```json Create payment payload
    {
      "order": {
          "items": [
          {
            "reference": "Sneaky NE2816-82",
            "name": "Sneaky",
            "quantity": 2,
            "unit": "pcs",
            "unitPrice": 25000,
            "taxRate": 0,
            "taxAmount": 0,
            "netTotalAmount": 50000,
            "grossTotalAmount": 50000
          }
        ],
        "amount": 50000,
        "currency": "SEK",
        "reference": "Order No 42369"
      },
      "checkout": {
        "integrationType": "EmbeddedCheckout",
        "url": "<WEBSHOP_URL>",
        "termsUrl": "<TERMS_URL>",
        "shipping": {
          "countries": [],
          "merchantHandlesShippingCost": true
        }
      }
    }
    ```

    Wenn die Eigenschaft `merchantHandlesShippingCost ` auf `true ` gesetzt ist, fährt Checkout erst dann mit dem Checkout fort, wenn Sie der Zahlung Versandkosten hinzufügen (siehe [Schritt 3](#build-step-3-calculate-and-update-shipping-cost-backend)). Doch zuerst müssen Sie Adressänderungen im Frontend verfolgen, um die neuen Versandkosten zu berechnen.

    <Warning>
    Diese Anleitung setzt voraus, dass Sie bereits wissen, wie Sie die Payment-API verwenden, um das Zahlungsobjekt für eine eingebettete Web-Integration zu erstellen. Falls nicht, lesen Sie zuerst den Leitfaden zur [Web-Integration](/web-integration/integrate-checkout-on-your-website-embedded).
    </Warning>
  </Step>
  <Step title="Auf das &quot;address-changed&quot;-Ereignis hören (Frontend)">
    Als Nächstes müssen Sie zusätzliche Funktionalität im Frontend-JavaScript implementieren, um eine Berechnung der Versandkosten auszulösen, wann immer sich die Adresse auf der Checkout-Seite ändert. Das Frontend sollte:

    1. **Auf das Ereignis** namens [`address-changed `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/checkout-js-sdk/#address-changed) **hören**.
    2. **Den Checkout einfrieren** mithilfe von Checkout JS, sobald ein Adressänderungsereignis ausgelöst wird. Dadurch wird das Checkout-Formular vorübergehend deaktiviert.
    3. Eine Anfrage an Ihr Backend senden, das neue Versandkosten berechnet und das **Zahlungsobjekt** bei der Nexi Group **aktualisiert**.
    4. **Den Checkout auftauen** mithilfe von Checkout JS. Dadurch wird das Checkout-Formular wieder aktiviert, sodass Ihr Kunde mit dem Checkout interagieren kann.

    Der folgende JavaScript-Code sendet eine Anfrage an Ihr Backend, damit neue Versandkosten berechnet und der Bestellung im Zahlungsobjekt hinzugefügt werden können:

    ### Auf das address-changed-Ereignis abonnieren

    <CodeGroup>

    ```javascript script.js
    // Frontend JavaScript
    // See Web integration guide for the complete JavaScript code

    // 1. Listen to the address-changed event
    checkout.on('address-changed', function (address) {
      if (address) {

        // 2. Disable checkout temporarily 
        checkout.freezeCheckout();

        // 3. Invoke backend to update shipping cost
        var request = new XMLHttpRequest();
        request.open("POST", 'update-shipping-cost.php');
        request.setRequestHeader("Content-Type", "application/json;charset=UTF-8");
        const body = {
            paymentId: paymentId,
            postalCode: address.postalCode,
            countryCode: address.countryCode
          };
          request.onload = function () {
            // 4. Don't forget to re-enable the checkout
            checkout.thawCheckout();
          }
          request.onerror = function () { 
            console.error('connection error'); 
          }
          request.send(JSON.stringify(body));
        }
    });
    ```

  <Step title="Versandkosten berechnen und aktualisieren (Backend)">
    Es ist an der Zeit, einen neuen HTTP-Endpunkt zur Aktualisierung der Versandkosten zu implementieren. Dieser sollte:

    1. Die Lieferadresse aus der HTTP-POST-Anfrage lesen, die im vorherigen Schritt vom JavaScript gesendet wurde.
    2. Neue Versandkosten basierend auf der Adresse berechnen.
    3. Die Versandkosten am Zahlungsobjekt mithilfe der Methode „update order“ der Payment-API hinzufügen oder ändern.

    <Warning>
    **Wichtig!**

    Wir benötigen für diese Aufgabe ein Backend-Skript, da wir die Payment-API aufrufen, die eine Server-zu-Server-Kommunikation erfordert. Sie dürfen den geheimen API-Schlüssel niemals vom Frontend-JavaScript an Checkout übergeben.
    </Warning>

    Lassen Sie uns den obigen Code durchgehen:

    1. Lesen Sie die Lieferadresse aus der HTTP-POST-Anfrage, die im vorherigen Schritt vom JavaScript gesendet wurde.
    2. Berechnen Sie neue Versandkosten basierend auf der Adresse und erzeugen Sie eine neue JSON-Payload mit den aktualisierten Versandkosten. Sie müssen die Methoden `computeShippingCost() ` und `updateOrder() ` definieren. Die von `updateOrder() ` zurückgegebene Payload sollte die vollständige Bestellung mit den aktualisierten oder neuen Versandkosten enthalten. Ein Beispiel-Payload mit Versandkosten ist unten dargestellt. Die API-Referenz enthält außerdem ein Beispiel für eine [Payload für die Update-Order-Anfrage](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-paymentid-orderitems-put-body).

  <Step title="Den Checkout-Ablauf testen">
    Es ist an der Zeit, die Funktionalität für die Verarbeitung der Versandkosten zu testen.

    1. Laden Sie die Checkout-Seite.
    2. Geben Sie Adressinformationen im Checkout-Formular ein.
    3. Wann immer die Adresse aktualisiert wird, sollte der JavaScript-Event-Handler ausgelöst werden und eine Anfrage an das Backend senden.
    4. Überprüfen Sie, ob die Checkout-Seite die Meldung "Versandkosten werden berechnet" anzeigt (wie im Screenshot in [Schritt 2](/add-shipping-cost/#build-step-2-listen-to-the-address-changed-event-frontend) gezeigt).
    5. Klicken Sie auf den "Bezahlen"-Button, sobald der Checkout reaktiviert wurde, und stellen Sie sicher, dass der Checkout-Ablauf keine Fehler enthält.

    <Info>
    Checkout stellt sicher, dass Sie Ihrer Bestellung Versandkosten hinzugefügt haben. Wenn Sie vergessen haben, die Eigenschaft [`costSpecified `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-paymentid-orderitems-put-body-shipping-costspecified) beim Aktualisieren der Bestellung auf `true ` zu setzen, wird nach dem Klick auf den "Bezahlen"-Button ein Fehler auf der Checkout-Seite angezeigt.
    </Info>

  </Step>

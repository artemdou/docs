## Überblick

Apple Pay mit Nexi Checkout

Schnelle, sichere Apple-Pay-Zahlungen in Ihrem Checkout, mit einer Einrichtung, die vom gehosteten bis zum vollständig eingebetteten Erlebnis skaliert.

Mit Apple Pay können Kunden Zahlungen in Sekundenschnelle mit ihren Apple-Geräten abschließen, ohne Kartendaten manuell eingeben zu müssen.

### Hauptvorteile:

- Ein-Tipp-Checkout mit Face ID oder Touch ID
- Keine Offenlegung oder Speicherung von Kartennummern auf Ihrer Website
- Optimiert für Mobilgeräte und Desktop
- Höhere Konversionsraten auf unterstützten Geräten

### Verfügbarkeit & Anforderungen:

Apple Pay wird nur dann automatisch angezeigt, wenn alle Bedingungen erfüllt sind:

- Für [Embedded Checkout](/apple-pay/#build-embedded-payment-page): Domain-Verifizierung
- Der Kunde verwendet ein unterstütztes Apple-Gerät
- Der Browser unterstützt Apple Pay (Safari empfohlen)
- Apple Pay ist in Ihrem Nexi-Konto aktiviert
- Mindestens eine unterstützte Kartenmarke ist aktiv
- Apple Pay erscheint nicht, wenn eine Anforderung fehlt; dies ist erwartetes Verhalten.
- Apple Pay wird nur unter iOS (16 oder höher) unterstützt. Unterstützte Browser sind: Safari, Firefox, Chrome und Edge.
- Apple Pay kann nicht aktiviert werden, ohne dass Visa/Mastercard/Dankort in Ihrem Konto vorhanden ist.

3. Klicken Sie auf **ENABLE APPLE PAY**, um die Apple-Pay-Option zu aktivieren.

## Embedded Payment Page

### Warum ist die Domain-Registrierung wichtig?

Apple verlangt eine Domain-Verifizierung, bevor Sie Apple-Pay-Zahlungen auf Ihrer Website akzeptieren können. Dieser Leitfaden erklärt den Prozess der Registrierung Ihrer Domains bei Apple Pay und der Aufrechterhaltung Ihres Domain-Verifizierungsstatus.

Die Registrierung verifiziert, dass Sie die Domain kontrollieren, und ermöglicht die Anzeige der Apple-Pay-Schaltfläche in Ihrem Checkout.

### Wichtige Punkte zu beachten

Bevor Sie die Domain registrieren, gibt es einige Dinge, die Sie wissen müssen:

- Alle Seiten, die Apple Pay enthalten, müssen über HTTPS bereitgestellt werden.
Ihre Domain muss ein gültiges SSL-Zertifikat haben.

- Wenn Ihre Domain vor öffentlichem Zugriff geschützt ist, müssen Sie diese für Apple Pay öffnen. Domains dürfen nicht hinter einem Proxy oder einer Weiterleitung liegen.

- Apple erwartet, die Datei an einem ganz bestimmten Ort auf Ihrer Domain zu finden. Erstellen Sie auf Ihrer Domain einen Speicherort mit der exakten URL ohne Weiterleitungen.

- Im Allgemeinen sollte sich die Datei apple-developer-merchantid-domain-association im Ordner „.well-known" und im öffentlichen Stammverzeichnis befinden, sodass sie über folgenden Link erreichbar ist:

  *https://[DOMAIN_NAME]/.well-known/apple-developer-merchantid-domain-association.*

- Es macht einen Unterschied, ob Sie die Domain-Datei mit „www" oder ohne „www" speichern. Stellen Sie sicher, dass Sie dieselbe Schreibweise der Domain im Platzhalter und für den Shop verwenden.

- Wenn Sie am Ende Ihrer Domain einen „/" haben, sollten Sie Ihre Datei auf einer Domain ohne diesen platzieren.

- Verwenden Sie die URL __ohne__ Schrägstrich am Ende.

  Beispiel *checkout.dibspayment.eu/.well-known/apple-developer-merchantid-domain-association*

## Übersicht der Plugin-Konfiguration

Um Embedded Checkout für Apple Pay zu aktivieren, müssen Sie Ihre Plugin-Umgebung und einen Speicherort für die __Domain-Verifizierungsdatei__ für die Transaktion vorbereiten.

Im Allgemeinen sollte sich die Datei __apple-developer-merchantid-domain-association__ im Ordner „.well-known" und im öffentlichen Stammverzeichnis befinden, sodass sie über folgenden Link erreichbar ist: __https://[DOMAIN_NAME]/.well-known/apple-developer-merchantid-domain-association__.

<Info>
Üblicherweise gibt es __public__-Ordner, die das Stammverzeichnis enthalten.
</Info>

__Nginx__ und __Apache__ sollten so strukturiert sein, dass diese Datei ohne Erweiterung aus dem Ordner „*.well-known*" gelesen werden kann.

Je nach Server und Plattform kann die Konfiguration unterschiedlich sein.

Hier sind einige Beispielkonfigurationen für Nginx- und Apache-Server, die wir möglicherweise zu den Konfigurationsdateien hinzufügen müssen:

### Nginx

Üblicherweise in `/etc/nginx/{shop}.conf `

```text Nginx code example
# .htaccess, .DS_Store, .htpasswd, etc., but keep .well-known available
location ~* /\.(?!well-known\/) {
deny all;
}
# files in .well-known should be served as plain text.
location ~* ^/\.well-known\/ {
default_type text/plain;
}
```

Weitere Informationen finden Sie unter folgendem [Referenzlink](https://devdocs.prestashop-project.org/8/scale/webservers/nginx/#allow-usage-of-well-known-directory-lets-encrypt--pki-validation-apple-pay).

### Apache

`.htaccess ` im public-Ordner

```text Apache code example
RewriteCond %{REQUEST_FILENAME} -f
RewriteRule
```

<Warning>
Es ist wichtig zu __beachten, dass dies generische Beispiele sind__, und jeder eine eigene Konfiguration haben kann, sodass es am besten ist, die Lösung an die oben genannten Anforderungen anzupassen.
</Warning>

Im Folgenden finden Sie Beispiele der jeweiligen Plattformen und Erläuterungen, wie die Grundkonfiguration angepasst werden kann.

## Benutzerführung

Der folgende Abschnitt beschreibt die Benutzerführung und den Zahlungsablauf für Ihren Kunden:

1. Wenn Ihr Verbraucher Apple Pay als Zahlungsmethode im Checkout auswählt, wird er zum Apple-Pay-Fenster geleitet.
2. Der Kunde kann zahlen mit
  - „__Pay with Touch ID__" oder
  - „__Double click to pay__", abhängig davon, welches System er verwendet.

## Zahlungsablauf

<Warning>
Wenn Sie den Parameter `paymentMethodsConfiguration ` verwenden, um nur die für die jeweilige Zahlung bevorzugten Zahlungsmethoden anzuzeigen, kann auch Apple Pay ausgewählt werden.  
Apple Pay ist nur auf Apple-Geräten verfügbar.
</Warning>

Apple Pay ist als Wallet definiert und kann allein und zusammen mit anderen Zahlungsmethoden angezeigt werden. Es wird nicht empfohlen, Apple Pay alleine anzuzeigen, wenn Sie das im Webshop verwendete Gerät nicht erkennen, bevor Sie die Zahlungsoptionen anzeigen.

Wenn Apple Pay alleine auf anderen Geräten als iOS/iPadOS/MacOS verwendet wird, sind keine sichtbaren Zahlungsmethoden vorhanden, was natürlich zu einer schlechten Benutzererfahrung führt. Apple Pay kann mit anderen Wallet-Zahlungsmethoden oder zusammen mit Karten kombiniert werden.

Empfohlene Verwendung des Parameters `paymentMethodsConfiguration ` für Apple Pay.

Wenn Sie mehr als eine Wallet konfiguriert haben, können Sie verwenden:

    `paymentMethodsConfiguration:[{"name":"Wallet"}]`

![Checkout Apple Pay Wallet](https://images.ctfassets.net/di680may5c0k/3A9DFBrjWF6T267ZP5CXDo/77273daedc9eeaab94838ed2d9f4fd1d/Easy_Apple_pay_wallet.png?w=323&h=700&q=50&fm=png)

Wenn Apple Pay als einzige Wallet verwendet wird, sollten Sie auch Karte einbeziehen:

    `paymentMethodsConfiguration:[{"name":"Card"},{"name":"ApplePay"}]`

## Bevor Sie beginnen

<Warning>
Bevor Sie beginnen, benötigen Sie eine Checkout-Seite, die die [Checkout-Testumgebung](/test-environment) verwendet.
</Warning>

## So testen Sie

Die Checkout-Seite bietet Ihren Kunden die Möglichkeit, per Rechnung zu bezahlen. Um die Rechnungsverarbeitung auf der Checkout-Seite zu testen, gehen Sie wie folgt vor:

## Riverty Nordic – Schritte

1. Navigieren Sie zu Ihrer Checkout-Seite
2. Wählen Sie „Pay later"
3. Geben Sie eine der **PINs** aus der untenstehenden Tabelle in das Feld „Personal identification number" ein.
4. Geben Sie die **Postleitzahl** und **Adresse** ein, die mit der ausgewählten PNO verknüpft ist.
5. Geben Sie einen beliebigen **Namen** und eine **Telefonnummer** ein
6. (Bedingt) Wenn eine starke Kundenauthentifizierung (SCA) angefordert wird, können Sie eine Test-BankID gemäß den [hier für SEK](https://www.bankid.com/en/utvecklare/test/skaffa-testbankid/test-bankid-get) verfügbaren Anweisungen verwenden. Verwenden Sie eine der Beispiel-PNOs aus der untenstehenden Tabelle.

<Warning>
Verwenden Sie einen geringen Betrag, um zu vermeiden, dass eine SCA für Testkäufe abgeschlossen werden muss.
</Warning>

## Riverty DACH – Schritte

1. Navigieren Sie zu Ihrer Checkout-Seite
2. Wählen Sie „Pay later"
3. Geben Sie eine beliebige **E-Mail** ein
4. Geben Sie die Testdaten aus der untenstehenden Tabelle für positive oder negative Testfälle ein: **Postleitzahl, Ländercode, Telefonnummer, Vorname, Nachname, Adresse, Land, Stadt und Geburtsdatum**
5. Akzeptieren Sie die Bedingungen
6. Klicken Sie auf die Schaltfläche **Pay**

<Warning>
Damit Riverty mit EUR im Demoshop angezeigt wird, müssen Sie wählen: *EasyInvoice, EasyInstallment, EasyCampaign* in der Zahlungseinstellung „Payment Type Filtering". Lesen Sie mehr dazu [hier](/payment/#build-payment-settings).
Sie müssen außerdem *MerchantHandlesConsumerData* deaktivieren.

Verwenden Sie einen geringen Betrag, um zu vermeiden, dass eine SCA für Testkäufe abgeschlossen werden muss.
</Warning>

## Before you start

<Warning>
This guide assumes that you have followed and implemented the guide [Integrate Checkout on your website](/web-integration). 
So, before you read this guide you should already have a checkout page up and running that handles single payments.
</Warning>

## Overview

**Subscription payments** (recurring payments) enable you to charge your customers on a regular basis, for example a monthly subscription for a product the customer must pay for every month.

A subscription is always associated with at least one payment object. A new subscription is created by the [Create payment](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post) method but with some [additional parameters](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post-body-subscription) that specify the terms for the subscription. Once a subscription has been created, you can charge the subscription. At every charge, a new payment object is created and then automatically charged. 

The subscription regulates how **often** and for **how long** you are allowed to create new payments. However, you are free to provide new order items and alter the amount for each individual payment.

![Subscription payments](https://images.ctfassets.net/di680may5c0k/5btC9LMoyphukrCQslsC0r/dd558833f31212fa62dcfbabb83fc8b9/subscriptions_overview.png?w=2014&h=1370&q=50&fm=png)

As the image above shows, once the subscription has been created, you can create new payments and charge the subscription without the need for additional interaction with the customer. 

<Warning>
It is important to create a payment with [unscheduled subscription object](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post-body-unscheduledsubscription) __amount=0__ to verify the card. 

If the card is valid, you will be able to query for the [token (unscheduled subscription ID)](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-paymentid-get) and use it for future charges.

This is the compliant way to register a card without charging the customer, and at the same time to be able to get a token back.
</Warning>

<Steps>
  <Step title="Create a new subscription">
    Assuming that you already have a checkout page that can manage single payments, the change required to create a subscription is straightforward.

    As already mentioned, this guide builds upon the [web integration guide](/web-integration) ([embedded](/web-integration/integrate-checkout-on-your-website-embedded) or [hosted](/web-integration/integrate-checkout-on-your-website-hosted)). To change the checkout to handle subscription payments rather than single payments, update the JSON request body that is passed when calling the [Create payment](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post) method. Here is an example of an updated version of `payload.json ` that includes the required `subscription ` properties:

    ```json payload.json
    {
        "order": {
            "items": [
                {
                    "reference": "10",
                    "name": "Monthly subscription",
                    "unit": "month"
                }
            ],
            "amount": 0,
            "currency": "SEK",
            "reference": "Subscription Test"
        },
        "checkout": {
            "integrationType": "EmbeddedCheckout",
            "url": "https://<YOUR_WEBSITE>/cart.html",
            "termsUrl": "https://<YOUR_WEBSITE>/terms.html"
        },
        "subscription": {
          "interval": 0,
          "endDate": "2021-07-18T00:00:00+00:00"
       }
    }
    ```

    <Info>
    Remember to replace the text `&lt;YOUR_WEBSITE&gt;` in the JSON request body.
    </Info>

    

    It is possible to specify an amount greater than zero and make an initial charge when creating the subscription (see the following step).

    <Info>
    **Troubleshooting**

    It is expected that __zero-amount__ is the correct setting in the __Create Payment request__. In case this setting does not pass and cards are getting declined when verifying, that is likely because the issuing bank doesn't allow zero-amount verifications. 

    To overcome this problem, please contact [Support](https://developer.nexigroup.com/nexi-checkout/en-EU/support/) and we will navigate you through this issue.
    </Info>

    Notice the subscriptions properties added to the end of the body:

    - The property [`subscription.interval `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post-body-subscription-interval) defines the minimum number of days between each recurring charge. In this case we use the value 0 which means no payment interval restrictions. 
    - The property [`subscription.endDate `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post-body-subscription-enddate) specifies the date that the subscription will expire. You are not allowed to charge the subscription after this date has passed.

    A successful request will return a JSON response similar to this:

    ```json Create subscription (response)

    {
      "paymentId": "0260000060003b7b47f0833960dc60d6"
    }
    ```

    The reponse object only contains a `paymentId ` which will be used throughout your customer's checkout flow. Once a valid payment card has been provided and your customer accepts the subscription, a new `subscriptionId ` will be created.
  </Step>
  <Step title="Make an initial charge (optional)">
    When you create a subscription, you have the option to make an initial charge at that time.

    ![Initial charge](https://images.ctfassets.net/di680may5c0k/4HIwOmMtHZToGkLNe4C5l7/4c72d2b2d26f3813290d1fab3c43ffc3/initial_charge.png?w=1736&h=714&q=50&fm=png)

    By setting the boolean property [`checkout.charge `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post-body-checkout-charge) to `true `, you tell Checkout that you want to charge the customer at the actual time when the subscription is created. The following JSON payload will create a new subscription and also automatically charge the customer for the specified order:

    ```json payload.json
    {
        "order": {
            "items": [
                {
                    "reference": "10",
                    "name": "Subscription product",
                    "quantity": 1,
                    "unit": "month",
                    "unitPrice": 10000,
                    "grossTotalAmount": 10000,
                    "netTotalAmount": 10000
                }
            ],
            "amount": 10000,
            "currency": "SEK",
            "reference": "Subscription Test"
        },
        "checkout": {
            "integrationType": "EmbeddedCheckout",
            "url": "https://<YOUR_WEBSITE>/cart.html",
            "termsUrl": "https://<YOUR_WEBSITE>/terms.html",
             "charge": true
        },
        "subscription": {
          "interval": 0,
          "endDate": "2021-07-18T00:00:00+00:00"
       }
    }
    ```
  </Step>
  <Step title="Test the subscription checkout">
    It's time to test the subscription checkout:

    1. Navigate to your checkout page in a web browser. If you have successfully created a subscription payment, the text on the payment button should read "Subscribe" rather than "Pay" (see the image below).
    2. Fill out the address form using your **email**, **phone**, and **postal code**.
    3. Insert any of the [sample card numbers](/test-card-processing#build-sample-credit-cards) and click the "Subscribe" button.

    ![Subscribe button](https://images.ctfassets.net/di680may5c0k/5ULQyXBb0sHQlgOTyIsoJ2/64c2b846e92fc1e32bb06be3d44cf616/subscription_screen_shot.png?w=2214&h=1130&q=50&fm=png)

    > info
    > 
    > It is possible to customize the text on the Subscribe button. See the  guide [Customize content](/customize-content/#build-customize-payment-button-text) for more information.

    <Info>
    **Troubleshooting** 

    In the example request body from step 1, an embedded integration is used. If you are using a hosted integration, you need to change the `checkout ` parameters in the request body to match a [hosted checkout](/web-integration/integrate-checkout-on-your-website-hosted).
    </Info>

    Once the subscription has been created, you need to obtain the subscription identifier. Typically you would persist all the subscription identifiers in your backend so that you can charge them on a regular basis later on. 

    There are two ways to retrieve information about a new subscription:
    - Find the subscription payments in Checkout Portal
    - Get payment and subscription details using the Payment API
  </Step>
  <Step title="View subscriptions details in Checkout Portal">
    After you have created a subscription payment, you can find the subscription details in Checkout Portal:

    1. Login to [Checkout Portal](https://portal.dibspayment.eu)
    2. Click ["Payments"](https://portal.dibspayment.eu/payments/NEW)
    3. Under "New" payments, your new subscription should be visible
    4. Click the newly created payment object
    5. Click "Transaction details"

    ![Verify subscription payment](https://images.ctfassets.net/di680may5c0k/26XyNMOPLDm7ijI3NjoHxh/aca4b73b8fa43e153c9e13ef0e8a80fd/Subscription_verify.png?w=2878&h=1296&q=50&fm=png)

    In the column named **RECURRING PAYMENT**, you should be able to see a subscription identifer. This identifier can be used when charging the subscription multiple times. You can also see the **frequency** (interval) and **end date** of the subscription.
  </Step>
  <Step title="Retrieve subscription using the Payment API">
    It is possible to retrieve information about a subscription programmatically using the Payment API. First retrieve the subscription payment object using the sample code below:

    ### Retrieve payment

    <CodeGroup>

    ```php retrieve-payment.php
    <?php

    $paymentId = "<YOUR_PAYMENT_ID>";
    $ch = curl_init('https://test.api.dibspayment.eu/v1/payments/' . $paymentId);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
    curl_setopt($ch, CURLOPT_HTTPHEADER, array(                                                                         
            'Content-Type: application/json',
            'Accept: application/json',
            'Authorization: <YOUR_SECRET_API_KEY>'));                                                
    $result = curl_exec($ch);
    $http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    echo "Response code: " . $http_code . PHP_EOL;
    $json_pretty = json_encode(json_decode($result), JSON_PRETTY_PRINT);
    echo $json_pretty . PHP_EOL;
    ```

    ```javascript retrieve-payment.js
    const http = require("https");

    const paymentId = "<PAYMENT_ID>";
    const secretKey = "<YOUR_SECRET_KEY>";

    const options = {
      "method": "GET",
      "hostname": "test.api.dibspayment.eu",
      "port": null,
      "path": "/v1/payments/" + paymentId,
      "headers": {
        "Authorization": secretKey
      }
    };

    const req = http.request(options, function (res) {
      const chunks = [];

      res.on("data", function (chunk) {
        chunks.push(chunk);
      });

      res.on("end", function () {
        const body = Buffer.concat(chunks);
        console.log(JSON.stringify(JSON.parse(body), null, 2));
      });
    });

    req.end();
    ```

    </CodeGroup>

    Remember to replace `&lt;PAYMENT_ID&gt;` with the payment identifier you received from the API when creating the subscription payment. The placeholder `&lt;YOUR_SECRET_KEY&gt;` should be replaced with your secret key. 

    A successful API call will return something similar to this:

    ```json Retrieve payment response
    {
      "payment": {
          "paymentId": "0260000060003b7b47f0833960dc60d6",
          "summary": {
              "reservedAmount": 10000
          },
          "consumer": {
            ...
          },
          "paymentDetails": {
              "paymentType": "CARD",
              "paymentMethod": "Visa",
              "invoiceDetails": {},
              "cardDetails": {
                  "maskedPan": "492500******0004",
                  "expiryDate": "1222"
              }
          },
          "orderDetails": {
              "amount": 10000,
              "currency": "SEK",
              "reference": "Subscription Demo Order with charge"
          },
          "checkout": {
              "url": "http:\/\/example.com\/cart.html"
          },
          "created": "2021-04-26T08:28:25.0946+00:00",
          "subscription": {
              "id": "4feb3b28d260458a88857b00f46261ba"
          }
      }
    ```

    The property `payment.subscription.id ` is the `subscriptionId ` you are looking for. Now you can use that identifier to get more information about the subscription, like this:

    ### Retrieve subscription details

    ```php retrieve-subscription.php
    <?php

    $subscriptionId = "<SUBSCIRPTION_ID>";

    $ch = curl_init('https://test.api.dibspayment.eu/v1/subscriptions/' . $subscriptionId);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
    curl_setopt($ch, CURLOPT_HTTPHEADER, array(                                                                         
            'Content-Type: application/json',
            'Accept: application/json',
            'Authorization: <YOUR_SECRET_API_KEY>'));                                                
    $result = curl_exec($ch);
    $http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    echo "Response code: " . $http_code . PHP_EOL;
    $json_pretty = json_encode(json_decode($result), JSON_PRETTY_PRINT);
    echo $json_pretty . PHP_EOL;
    ```

    Again, make sure you replace all the placeholder strings marked with `<...>`. 

    The JSON response from the retrieve subscriptions request can look something like this:

    ```json Subscription details response
    {
        "subscriptionId": "205e808f6fba40828b0389c8b943aefe",
        "frequency": 0,
        "interval": 0,
        "endDate": "2021-07-18T00:00:00.0000+00:00",
        "paymentDetails": {
            "paymentType": "CARD",
            "paymentMethod": "Visa",
            "cardDetails": {
                "expiryDate": "1222",
                "maskedPan": "492500******0004"
            }
        }
    }
    ```

    Now you know how to retrieve the subscription identifier and other details related to a subscription payment. The subscription identifier is required when you want to charge a subscription at a later stage. You can charge multiple subscriptions in bulk using the API method [Bulk charge subscriptions](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-subscriptions-charges-post).

    > tada
    > **Congratulations!**
    > 
    > Now you have a checkout page that supports subscription payments and you also know how to retrieve the subscription identifiers required for future charges.
  </Step>
</Steps>

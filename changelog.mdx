See more details on express buttons /express buttons i" }}>
      ### New Features

      - Express button for Google Pay and Apple Pay
       - Merchants can now add Google Pay and Apple Pay express button directly on the product page, enabling customers to pay instantly. See more details on [express buttons](/express-buttons-in-product-page).
      - Vipps now available outside Norway
       - Merchants in Denmark, Sweden, Germany, and Austria can now offer Vipps without needing Norwegian VAT registration or a separate Norwegian MID. See more details on [Vipps](/vipps).
      - Payment Account Reference (PAR) visibility
       - Payment Account Reference (PAR) makes it possible to link multiple card numbers back to a single underlying customer account, giving merchants a unified view of customer activity. This enables seamless tracking of the full customer journey across both online and instore channels.
       - Links to webhooks: [Get Payment Responses](https://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-paymentid-get-responses) and [Payment Reservation Created (v2)](https://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-webhooks-v1/#paymentreservationcreatedv2). 
       - Merchants can now access PAR data to link multiple cards to one customer account. 
       - Please observe: PAR is available only upon request (please contact [support](https://developer.nexigroup.com/nexi-checkout/en-EU/support/)).

      ### Improvements

      - Available payment methods based on country code
       - Improvement on how payment methods are displayed in the checkout. Changes have been made based on shopper country code and merchant-defined country settings, which makes the experience more consistent and relevant.
      - Real card number for wallet transactions in Easy Portal
       - Wallet transactions now display the actual card number, rather than only tokenized or virtual ones, allowing merchants to search using the last four digits and more easily locate transactions.

      ### Bug fixes

      - Tustly is hidden when mandatory consumer data is missing
       - If the Checkout window lacks required consumer data such as email or name then Trustly is no longer shown as an option. This option avoids a broken payment path and surfaces a clear error instead.
      - RatePay device fingerprinting reliability improved
       - Fingerprinting now runs more consistently, improving the ability to retrieve information for a device token and reducing execution failures.
    </Update>

      We would like to inform you that Nets Easy / Nexi Checkout will change its banking partner from SEB to Danske Bank effective December 16th, 2025.

      ### What does this entail?
      This means that, as of December 16th, all bank transfers to your Easy / Nexi Checkout account (the account used to ensure the completion of initiated refunds), must be made using the updated account details available in the Easy / Nexi Checkout Portal.
      - The account numbers from Danske Bank will be available in the Easy / Nexi Checkout Portal.
      - Always use your Reference ID when making a transfer.
      - You can find your Reference ID in the Portal: [https://portal.dibspayment.eu/](https://portal.dibspayment.eu/) in the menu: Payouts → +ADD FUNDS.

      ### What do you need to do?
      If you have stored the old SEB bank account details internally, we kindly ask you to take note of the above information, regarding the new bank account, and update them accordingly.
      Please note that this change only applies to the bank account used for processing refunds on your Easy / Nexi Checkout agreement.

      ### Date & Time
      The change will be effective as of December 16th 2025.

      If you have any questions about this, then you can contact Support from [here](https://developer.nexigroup.com/nexi-checkout/en-EU/support/).

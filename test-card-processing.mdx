## Before you start

<Warning>
Before you start, you need a checkout page that uses the [Checkout test environment](/test-environment).
</Warning>

## How to test

The Checkout page provides your customers with the option to pay by card. To test card payments on the checkout page, follow the steps below:

1. Navigate to your checkout page.
2. Select "Pay by card".
3. Add one of the sample card numbers from **Number** column in the table below.
4. Enter an expire date in the future.
5. Enter any 3 digits in the CVC field.
6. Enter any arbitrary name, address, and phone number in the address form.

![Test card processing](https://images.ctfassets.net/di680may5c0k/5l8yIooGbWH44h9PO533Od/91649112157f5a51a0d29035b1a8d9e7/Screenshot_2021-03-30_at_21.37.59.png?w=1676&h=1168&q=50&fm=png)

<Warning>
When using the sample cards in the test environment, some amounts have special meaning and will generate error codes or increase the API response times. This is not a bug, but a feature of the test environment so that you can test your system against error codes and delayed responses. See the following section for more information.
</Warning>

## Amounts with special meaning

It is possible to test how your website behaves when Checkout returns various **error codes** or if the checkout process is **delayed**. The following amounts have a special meaning and changes the checkout behaviour when creating a new payment in the test environment:

- The amount **0.XX** and **1.XX** will raise the **error code XX**. For example, if the property [`order.amount `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post-body-order-amount) is set to `133`, the error code `33` will be raised.

- The amount **3.XX** will **delay** the API response by **XX seconds**. For example, if the property [`order.amount `](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post-body-order-amount) is set to `310`, the reply from the checkout flow will be delayed by 10 seconds.

<Info>
Note that amounts are specified in the [lowest monetary unit for the given currency, without punctuation marks](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api#data-types-and-formats).
</Info>

You can read more about reason codes from [here](/reason-codes).

## Discounts

It is possible to specify for discounts in the payment requests. This is done by adding an order line with a negative amount.

You can see an example of this in the payload example from [here](/web-integration/integrate-checkout-on-your-website-embedded/#build-step-2-create-a-payment-object-backend).

<Warning>
Please note that it is not possible for the total amount to ever be 0 or lower for the following requests:
- [unit price](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post-body-order-items-unitprice)
- [gross total amount](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post-body-order-items-grosstotalamount)
- [net total amount](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-post-body-order-items-nettotalamount)
- [orderItems](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-payments-paymentid-charges-post-body-orderitems)
- [charge orderItems  ](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1/#v1-charges-chargeid-refunds-post-body-orderitems)
</Warning>

If the [Create payment request](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/payment-v1#v1-payments-post) contains `"charge":true `, then charge with test card 4925000000000079 will not fail.
</Warning>

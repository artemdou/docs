## What is the test environment?

The test environment allows you to perform test purchases using card and invoice payments. The test environment comprises:

- [Integration keys](/access-your-integration-keys) for testing
- [Sample card numbers](/test-card-processing)
- [Sample invoice addresses](/test-invoice-installment-processing)
- Test [API](https://developer.nexigroup.com/nexi-checkout/en-EU/api) available at the base address: ```https://test.api.dibspayment.eu ```
- [Checkout.js](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/checkout-js-sdk) available at: ```https://test.checkout.dibspayment.eu/v1/checkout.js?v=1```

## Using the test environment

In order to set up a checkout page that are using the test environment, you need to:

1. Send all the API requests toward the [test API base address](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api#environments-and-base-addresses).
2. Use the [integration keys](/access-your-integration-keys) for testing.
3. Load the correct version of the [Checkout JS SDK](https://developer.nexigroup.comhttps://developer.nexigroup.com/nexi-checkout/en-EU/api/checkout-js-sdk) (if you are building an embedded checkout).

Once you have the checkout page configured toward the test environment, you can test card and invoice processing using the sample cards and sample invoice addresses.

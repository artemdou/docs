## Before you start

<Warning>
Before you start, you need a checkout page that uses the [Checkout test environment](/test-environment).
</Warning>

## How to test

The Checkout page provides your customers with the option to pay by invoice. To test invoice processing on the checkout page, follow the steps below:

## Riverty Nordic steps

1. Navigate to your checkout page
2. Select "Pay later"
3. Add one of the **PINs** from the table below in the field named "Personal identification number".
4. Enter the **Postal code** and **Address** associated with the PNO you selected.
5. Enter an arbitrary **name** and **phone number**
6. (Conditionally) If strong consumer authentication (SCA) is requested then you are able to use a test BankID using the instructions found [here for SEK](https://www.bankid.com/en/utvecklare/test/skaffa-testbankid/test-bankid-get) payments. Make sure to use one of the sample PNO found in the table below.

<Warning>
Use a small amount in order to avoid having to complete SCA for test purchases.
</Warning>

## Riverty DACH steps

1. Navigate to your checkout page
2. Select "Pay later"
3. Enter an arbitrary **email**
4. Enter in the test data from either the positive or negative test case tables below: **postal code, country code, phone number, first name, last name, address, country, city and date of birth**
5. Accept the terms
6. Click on the **Pay** button

<Warning>
In order to have Riverty be shown with EUR in the demostore, you have to choose: *EasyInvoice, EasyInstallment, EasyCampaign* in the Payment Type Filtering payment setting. Read more about this from [here](/payment/#build-payment-settings). 
You also need to toggle off *MerchantHandlesConsumerData*.

Use a small amount in order to avoid having to complete SCA for test purchases.
</Warning>
